### 안정 ID 규약 (Q2-A 결정)

```
카테고리 prefix 1자리 + 하이픈 + 2자리 순번

F-NN : 기능 (Feature)
E-NN : 엣지 (Edge)
R-NN : 에러 (erRor) — 엣지 E와 충돌 회피
T-NN : 테스트 (Test)
U-NN : UI·UX (Ui)
G-NN : 회귀 (reGression) — 에러 R과 충돌 회피
P-NN : 플랫폼 (Platform)
```

- 순번은 **카테고리 내부에서 01부터** (예: `F-01`, `F-02`, `E-01`, ...)
- ID 중복/누락/포맷 위반은 step-8 파싱 실패로 처리 (아래 "파싱 실패 정책" 참조)
- Grep 앵커: `- \[([ x~\-])\] ([FERTUGP])-\d{2}\b`

### 4상태 모델 (Q3-B 결정)

| 상태 | 기호 | 의미 | 사유 | 갱신 주체 |
|------|------|------|------|-----------|
| 미완료 | `- [ ] F-01 ...` | 기본값 | - | - |
| 완료 | `- [x] F-01 ...` | 실측 통과 | - | step-8에서 사용자 보고 기반 Claude가 갱신 |
| 연기 | `- [~] deferred (다음 마일스톤에서 벤치마크 예정) F-02 ...` | 필요하지만 이번엔 보류 | **필수** | 사용자 수동 또는 사용자 명시 지시 시 Claude 갱신 |
| 해당없음 | `- [-] N/A (백엔드만 건드려 UI 검증 불필요) F-03 ...` | 서브태스크 범위 밖 | **필수** | 사용자 수동 또는 사용자 명시 지시 시 Claude 갱신 |

- `[~]`/`[-]`는 **사유 필수**. 사유 없는 경우 파싱 실패
- `[x]`는 step-8에서 Claude가 사용자 보고 받으면 갱신. AI 독단 `[x]` 금지
- `[~]`/`[-]` 전환 판단은 Claude가 추측하지 않음 — 사용자 명시 사유가 있어야 적용

### 섹션 포맷 (안정 ID + 4상태 반영)

```markdown
## Feature List
<!-- size: 중형 | count: 18 | skip: false -->

### 기능
- [ ] F-01 로그인 후 피드 진입 시 스켈레톤 UI 표시
- [ ] F-02 스크롤 무한로딩 정상 동작

### 엣지
- [ ] E-01 빈 피드 상태 메시지 노출

### 에러
- [ ] R-01 네트워크 실패 시 재시도 버튼 표시

### 테스트
- [ ] T-01 Vitest 단위 테스트 통과

<!-- 조건부 카테고리 (해당 시만) -->
### UI·UX
- [ ] U-01 다크모드 색상 대비 확인
```

- **HTML 주석 메타데이터**: `size` (소형/중형/대형), `count` (총 체크박스 수), `skip` (소형 스킵 모드 여부) — **한 줄 단일 메타가 SSOT** (02/향후 8.2 모두 이 메타만 파싱)
- **카테고리는 H3** (`### `), 순서 고정: 기능 → 엣지 → 에러 → 테스트 → (UI·UX) → (회귀) → (플랫폼)
- **체크박스 기호 집합**: `[ ]` / `[x]` / `[~]` / `[-]` (4상태)
- **사유 필드**: `[~]`/`[-]` 뒤에 `(사유)` 형식, ID 앞에 위치 — 예: `- [~] deferred (다음 마일스톤) F-02 ...`
- 소형 스킵 모드: `<!-- size: 소형 | count: 4 | skip: true -->` — 체크박스 최소 3~5개만 생성

### 크기 판정 규칙 (위험도 예외 포함)

```
1차: 변경 파일 수
  - 1~2개  → 소형
  - 3~5개  → 중형
  - 6개 이상 → 대형

2차: 예외 규칙 (1차 결과를 한 단계 올림)
  - 단일 파일 500라인+ 수정 → 소형이어도 중형
  - 상태 관리 / 인프라 / 마이그레이션 / 상태머신 → 중형 이상 강제
  - 공통 컴포넌트 / 공용 라이브러리 변경 → 중형 이상 강제

사용자 확인: AI가 1차+2차 적용 결과를 A/B/C로 제안, 사용자 확인
  예: "변경 파일 2개, 단일 파일 800라인 수정 → B(중형, 15~25) 제안. A(소형) / B(중형, 추천) / C(대형)?"
```

### 파싱 실패 정책 (Q2-A 규약 보강)

step-8(02) 및 향후 §8.2가 공유하는 파서는 다음 경우를 **파싱 실패**로 처리:

| 실패 유형 | 감지 기준 | 동작 |
|-----------|-----------|------|
| ID 포맷 위반 | prefix 미일치 / 순번 2자리 아님 / 중복 ID | 차단 + "canonical formatter로 재생성" 제안 |
| 상태 기호 위반 | `[ ]` / `[x]` / `[~]` / `[-]` 외 기호 | 차단 + 허용 기호 안내 |
| `[~]`/`[-]` 사유 누락 | 사유 괄호 없음 또는 빈 괄호 | 차단 + 사유 보정 요청 |
| HTML 메타 누락 | `<!-- size: ... -->` 없음 or 필드 부족 | 차단 + 메타 복구 제안 |
| 카테고리 순서 위반 | H3 순서 고정 규칙 위반 | 경고만 (파싱은 진행) |

**"파싱 실패 = 통과" 금지**. Silent pass는 8.2 태스크에서 차단 로직 도입 시 치명적 구멍.

### 항목 생성 컨텍스트 (초안 품질)

AI가 각 항목을 유추할 때 다음을 **모두** 컨텍스트로 읽는다:
- 서브태스크 `유형` (추가기능 / 버그수정 / 리팩토링 / 문서업데이트)
- `In-Scope` 블릿
- `변경 파일` 표의 각 파일·변경 유형·설명
- (참고) `Out-of-Scope` — 역검증용
- 프로젝트 룰 (`.claude/rules/*-scope.md`) — 필수 검증 명령, 코드 스타일 규칙

### 카테고리 분배 가이드 (균등 편중 방지)

중형(15~25개) 기준 권장 분배 비율:
- 기능 F: 30~40%
- 엣지 E: 15~20%
- 에러 R: 10~15%
- 테스트 T: 20~25%
- (조건부) UI·UX U / 회귀 G / 플랫폼 P: 해당 시 각 5~15%

AI 초안 생성 시 "기능 20개 + 엣지 0개" 식 편중 방지. 실제 서브태스크 유형에 따라 조정 가능 (예: 문서업데이트 서브태스크는 테스트 T 비율 낮춤).

## 리뷰 결과

### Claude 리뷰 (문서 일치성)
- **의존성 그래프 붕괴**: 메인태스크 서브태스크 표는 02·03이 01에 의존한다고 선언하지만, 01 파서 규약(H3 + HTML 메타)만으로는 02의 `§카테고리 #번호` 매핑·03의 `skip` 판정이 일의적으로 파생되지 않음. 01이 SSOT라면 안정 식별자 규약이 먼저 확정돼야 함. → **재작업으로 해소**: 안정 ID 규약(F-NN, E-NN 등) + 4상태 모델 + 파싱 실패 정책 모두 01에 명시.
- **"Out-of-Scope: 기존 서브태스크 문서 retrofit"**: 문구 모호. 이번 G1 구현 서브태스크(01~04) 자체가 Feature List 없이 기존 방식으로 진행되는 개밥 먹기 모순 → 완료 조건 명시 추가.
- **카테고리 분배 규칙 부재**: 15~25개를 7종에 균등/비균등 분배할지 명시 없음 → **재작업으로 해소**: 카테고리 분배 가이드 추가.

### Codex 리뷰 (기술적 타당성)
- ⚠️ **조건부 승인** → 재작업으로 승인 조건 충족
- 크기 판정 순서 불안정 → **재작업으로 해소**: 변경 파일 표 선 확정 → 크기 판정 순서 고정
- 안정 ID 규약 필요 → **재작업으로 해소**: `F-NN`/`E-NN`/... 카테고리 prefix 규약 도입
- 테스트 전부 수동 → **재작업으로 해소**: 포맷 fixture 3종 파싱 검증 추가

### 구멍 찾기 리뷰 (critical-review)
- **치명 1건**: 파서 SSOT 실패 정책 없음 → **재작업으로 해소**: 파싱 실패 정책 표 추가 ("파싱 실패 = 차단 + 복구 제안", silent pass 금지)
- **중대 1건**: 파일 수 단일 축 크기 판정이 위험도 왜곡 → **재작업으로 해소**: 2차 축(런타임 위험) 예외 규칙 추가 (단일 파일 상태관리/인프라/마이그레이션 → 중형 이상 강제)

### 최종 결정: ✅ **승인** (재작업 반영 완료)
