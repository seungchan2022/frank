## 요구사항 (인터뷰 합의)

| 항목 | 결정 |
|------|------|
| 네비게이션 | FeedView 기존 NavigationStack에 navigationDestination 추가 (push/pop) |
| content 필드 | 미사용 — summary + insight만 표시, 원문은 Safari 링크 |
| 레이아웃 | 카드 없이 자유 배치 (제목 → 출처/날짜 → 요약 → 인사이트 → 원문 링크) |

| 에러 처리 | 에러 메시지 + 재시도 버튼 |

## 서브태스크

### ST-1: ArticleDetailFeature (신규)

- `@Observable @MainActor` + 플랫 프로퍼티 패턴
- `init(articleId: UUID, articlePort: any ArticlePort)`
- Action: `.loadArticle`
- 상태: `article: Article?`, `isLoading`, `errorMessage`
- 상태 전이 헬퍼: `beginLoading()`, `failLoading()`
- 테스트 7개: 초기 상태, 성공, 실패, 재시도 에러 초기화, callCount, 올바른 ID, summary nil

### ST-2: ArticleDetailView (신규)

- ScrollView 자유 배치: 제목 → 출처/날짜 → AI 요약 → 핵심 인사이트 → 원문 보기
- `.task { await feature.send(.loadArticle) }`
- 로딩: 중앙 ProgressView
- 에러: 아이콘 + 메시지 + "다시 시도" 버튼
- 원문 보기: `.borderedProminent` 버튼 + Safari openURL
- `ArticleCardView.relativeTimeText()` 재사용

### ST-3: FeedView 네비게이션 연결 (수정)

- `ArticleCardView.swift`: Button+openURL 제거 → 표시 전용 VStack
- `FeedView.swift`: `articlePort` 파라미터 추가, NavigationLink(value: article.id) + navigationDestination
- `FrankApp.swift`: FeedContainerView에서 articlePort를 FeedView로 전달
