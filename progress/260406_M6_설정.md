# M6: 설정 (SettingsFeature)

> 생성일: 260406
> 상태: ✅ 완료

## 목표

SettingsPlaceholderView를 완전한 SettingsFeature로 교체. 태그 관리 + 로그아웃 기능.

## 요구사항

| 항목 | 결정 |
|------|------|
| 진입 방식 | 피드 네비바 gear → sheet (현행 유지) |
| 태그 편집 | 설정 리스트 → "태그 관리" → push TagManagementView |
| 태그 최소 개수 | 최소 1개 (마지막 태그 해제 방지) |
| 피드 동기화 | 태그 저장 시 sheet 즉시 닫고 직접 reloadAfterTagChange 호출 |
| 로그아웃 | 확인 Alert 추가 |
| 추가 항목 | 없음 (태그 관리 + 로그아웃만) |
| DB 마이그레이션 | 제외 (profiles 테이블 이미 존재) |

## 구현 파일

| 파일 | 설명 |

| `Features/Settings/SettingsFeature.swift` | @Observable Feature — loadTags, toggleTag, saveTags, signOut |

## 아키텍처 결정

- **SettingsFeature는 매번 새로 생성**: sheet 열 때 인스턴스를 새로 만들어 상태 초기화
- **sheet(item:) 사용**: Identifiable 바인딩으로 첫 오픈 시 빈 화면 방지
- **onTagsSaved 직접 reload**: onDismiss 타이밍 이슈 회피, settingsFeature=nil + Task로 직접 호출
- **전체 탭 클라이언트 필터링**: 서버가 tagId=nil 시 모든 기사 반환 → 현재 구독 태그로 필터링
- **최소 1개 보장**: toggleTag에서 `selectedIds.count > 1` 가드

## 버그 수정 이력

| 이슈 | 원인 | 해결 |
|------|------|------|
| 태그 저장 후 피드 새로고침 안 됨 | onDismiss 콜백 타이밍 이슈 | onTagsSaved에서 직접 reload 호출 |
| 전체 탭에 이전 태그 기사 표시 | 서버가 모든 기사 반환 | fetchAndCache + loadMore에 클라이언트 필터링 |
| 처음 sheet 열면 빈 화면 | isPresented + if let 타이밍 | sheet(item:) 바인딩으로 변경 |
