## S1: HTTP retry/size-limit 공통 유틸리티 모듈

**파일**: `server/src/infra/http_utils.rs` (신규), `server/src/infra/mod.rs` (pub mod 추가)

**산출물**:
- `RetryConfig` 구조체: `max_retries`, `base_delay_ms`, `max_response_size`, `retryable_statuses`
- `send_with_retry(builder_factory, client, config)` 함수: RequestBuilder 팩토리 클로저를 받아 retry + size-limit 체크 수행. 반환: `Result<reqwest::Response, AppError>`
- retryable 상태코드: 429, 500, 502, 503, 504 + timeout/connection reset
- 호출측에서 `.json()` 파싱 담당

**테스트** (`tokio::time::pause()` 사용하여 실제 대기 없이 검증):
- retry 성공 (2회 실패 후 3회째 성공)
- retry 전부 실패 시 마지막 에러 반환
- backoff 지수 증가 검증 (`tokio::time::advance()`)
- non-retryable 상태코드는 즉시 반환
- size-limit 초과 시 에러
- size-limit 이하 시 통과
- Content-Length 헤더 없으면 통과 (검증 스킵)
- `max_retries=0` 경계 케이스: retry 없이 즉시 반환
- `Content-Length == limit` 경계 케이스: 통과 (초과만 거부)

**dev-dependencies**: `Cargo.toml`에 `wiremock` 추가

## S8: 웹 Auth 구독 해제 구현 + 테스트

**파일**:
- `web/src/lib/stores/auth.svelte.ts` — 핵심 수정
- `web/src/routes/+layout.svelte` — cleanup 연결
- `web/src/lib/stores/auth.test.ts` — 테스트 추가

**변경** (B안: store 레벨 subscription 관리):
- `auth.svelte.ts`에 모듈 레벨 `subscription` 변수 + `initialized` boolean 플래그 추가
- `initAuth()` 중복 호출 방지: `if (initialized) return;` → `initialized = true;` (await 전에 설정하여 race condition 방어)
- `initAuth()` 실패 시 `initialized = false`로 롤백 (try-catch)
- `cleanupAuth()` 함수 추가: subscription null 체크 후 unsubscribe + 상태 초기화 + `initialized = false` (idempotent)
- `+layout.svelte`의 `onMount`에서 cleanup 함수 반환

**테스트** (`beforeEach`에서 `cleanupAuth()` 호출하여 모듈 상태 리셋):
- 중복 initAuth() 호출 시 onAuthStateChange 1회만 호출
- cleanupAuth() 호출 시 unsubscribe 1회 호출
- cleanupAuth() 중복 호출 시 에러 없음 (idempotent)
- cleanup 후 콜백 실행 무효 (상태 변경 안 됨)
- getSession() 실패 시 loading=false, session=null 유지 + initialized 롤백
