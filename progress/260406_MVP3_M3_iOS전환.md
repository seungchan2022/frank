## 핵심 원칙 (절대 위반 금지)

1. **격리**: ios/ 디렉토리만 수정. server/와 web/는 절대 건드리지 말 것
2. **Mock-First**: ST-3까지는 외부 자원(Supabase, Rust 서버) 호출 0건 유지
3. **M1.5 인프라 강제 사용**: `ios/Frank/Frank/Sources/Core/Adapters/Mock*.swift`를 우회하지 말 것. Mock 어댑터는 영구 보존
4. **기존 Supabase 어댑터 보존**: `SupabaseTagAdapter`, `SupabaseArticleAdapter`는 **삭제 금지**. Mock과 함께 공존시킴 (M2처럼 RealClient 내부만 교체하는 패턴이 iOS에도 적용)
5. **공통 문서 손대지 말 것**: `progress/` 폴더에서 본 문서(`260406_MVP3_M3_iOS전환.md`)와 회고만 수정. 로드맵, M2 문서, M1.5 문서는 read-only
6. **Server 작업 발견 시 즉시 정지**: server/ 변경이 필요해 보이면 작업 멈추고 사용자에게 보고. 메인 탭에서 hotfix 처리
7. **Tuist 주의**: tuistenv 회피 위해 `~/.tuist/Versions/4.31.0/tuist`를 직접 호출

## 작업 흐름

```
1. iOS 모델 미스매치 정정 (Profile, Article, Tag) + MockFixtures 갱신
   ↓
2. Mock 모드로 화면/네비/상태 동작 확인 (외부 호출 0)
   ↓
3. APITagAdapter / APIArticleAdapter 신설 (TDD, URLSession + Rust API)
   ↓
4. SupabaseAuthAdapter.updateOnboardingCompleted 정정 (Rust PUT /api/me/profile 호출)
   ↓
5. AppDependencies.live() 어댑터 교체 (Supabase 어댑터는 보존, 사용 안 할 뿐)
   ↓
6. MVP2 부채 흡수 (LoadingPhase enum, UITest 보강, 태그 로딩 중복 추출)
   ↓
7. XCUITest E2E + 시뮬레이터 검증
   ↓
8. 머지 준비 + main rebase
```

**M2(웹)와 완전 독립**. 두 worktree 탭에서 동시 작업 시 외부 자원 공유 없음.
