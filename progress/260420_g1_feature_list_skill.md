## 요약

하네스 비교분석 결과 최우선 미구현 항목으로 확정된 **G1 (Feature List 자동 생성)**을 `/step-4` 인터뷰 직후 자동 생성 단계로 도입한다. 서브태스크별 검증 체크리스트(기능/엣지/에러/테스트 + 조건부 UI·UX/회귀/플랫폼)를 15~25개 기준으로 생성하고, `/step-8`에서 사용자가 한 항목씩 순회 검증하며 결과 보고에 따라 상태를 갱신한다. Anthropic 실패 4유형 중 ①(대충 끝내려 함)·②(다 했다고 거짓말)를 체계적으로 막는다.

> **범위 축소 (2026-04-20, 리뷰 후)**: 본 G1 사이클은 **01(step-4 생성) + 02(step-8 순회 검증) + 04(하네스 문서 동기화)** 로 한정. 커밋 차단 로직(구 03)은 분석 문서 `§8.2` 영역으로 확인돼 **별도 후속 태스크로 이관**. 기존 03 문서는 §8.2 태스크 착수 시 참고용으로 `progress/subtask/260420_g1_feature_list_skill/03-step9-commit-block.md`에 그대로 남겨둔다.
