# MVP10 M1: iOS 퀴즈 UX 완성 + 즐겨찾기 배지 미표시 수정

> 생성일: 2026-04-14
> 마일스톤: MVP10 M1
> 상태: 완료 — feature/260414_quiz_ux_favorites_badge)
> 예상 기간: 1일
> 의존성: 없음 (서버 API quiz_completed 반환 이미 구현됨)

## 목표

1. iOS 퀴즈 완료 화면에 "다시 풀기" + "오답 보기" 버튼 추가 (웹과 동일한 UX)
2. 퀴즈 완료 후 즐겨찾기 배지(`퀴즈 ✓`)가 즉시 반영되도록 로컬 상태 동기화 수정

## 버그 원인 요약

```
QuizView.nextQuestion() → onQuizCompleted?() 호출
→ ArticleDetailView: quizFeature.markQuizCompleted()
→ QuizFeature: FavoritesPort 직접 호출 → 서버 DB만 업데이트
→ FavoritesFeature.items 배열은 갱신되지 않음  ← 버그 위치
→ FavoritesFeature.isQuizCompleted(url) → false (stale cache)
→ FavoriteRowView.quizCompleted 배지 미표시
```

핵심: `QuizFeature`와 `FavoritesFeature`가 각각 별도 `FavoritesPort` 인스턴스를 사용해
서버는 업데이트되지만 UI 상태(`FavoritesFeature.items`)가 갱신되지 않음.

### ST-2: FavoritesFeature.markQuizCompleted(url:) 구현

- **파일**: `ios/Frank/Frank/Sources/Features/Favorites/FavoritesFeature.swift`
- **작업**:
  - `markQuizCompleted(url:)` 메서드 추가
  - `addFavorite` / `removeFavorite` 패턴 동일하게 적용
  - `FavoriteItem`은 `struct` + `let` 프로퍼티 → `map`으로 불변 복사 갱신

- **구현 의사코드**:
  ```swift
  func markQuizCompleted(url: String) async {
      operationError = nil
      do {
          try await favorites.markQuizCompleted(url: url)
          items = items.map { item in
              guard item.url == url else { return item }
              return FavoriteItem(
                  id: item.id, userId: item.userId, title: item.title,
                  url: item.url, snippet: item.snippet, source: item.source,
                  publishedAt: item.publishedAt, tagId: item.tagId,
                  summary: item.summary, insight: item.insight,
                  likedAt: item.likedAt, createdAt: item.createdAt,
                  imageUrl: item.imageUrl, quizCompleted: true
              )
          }
      } catch {
          operationError = "퀴즈 완료 기록에 실패했습니다."
      }
  }
  ```

- **TDD 체크포인트**: ST-1 테스트 3개 모두 통과 확인

### ST-3: QuizView 완료 화면 버튼 추가

- **파일**:
  - `ios/Frank/FrankTests/Features/Quiz/QuizViewSessionTests.swift` (테스트 추가)
  - `ios/Frank/Frank/Sources/Features/Quiz/QuizView.swift` (구현)

- **테스트 추가**:
  - `wrongAccumulator.items.isEmpty == false 일 때 오답 보기 버튼 활성화 조건 확인`
  (QuizView는 SwiftUI View라 직접 단위테스트 어려움 — 상태 로직은 SessionWrongAnswerAccumulator 레벨에서 검증)

- **QuizView 변경사항**:
  1. `onRetry: (() -> Void)? = nil` 프로퍼티 추가
  2. `finishedView`에 버튼 2개 추가:
     - **"다시 풀기"**: `onRetry != nil`일 때만 표시, 탭 시 상태 원자적 리셋 후 `onRetry?()` 호출
     - **"오답 보기"**: `!wrongAccumulator.items.isEmpty`일 때만 표시, 탭 시 오답 시트 (`sheetWrongAnswers`) 전달 방식 또는 `onShowWrongAnswers` 콜백 활용

- **상태 리셋 7개 원자적 수행** ("다시 풀기" 탭 시):
  ```swift
  currentIndex = 0
  selectedIndex = nil
  confirmed = false
  score = 0
  finished = false
  quizCompletedMarked = false
  wrongAccumulator.reset()
  onRetry?()
  ```

### ST-4: ArticleDetailView 연결 수정 + QuizFeature 정리

- **파일**:
  - `ios/Frank/Frank/Sources/Features/Detail/ArticleDetailView.swift`
  - `ios/Frank/Frank/Sources/Features/Quiz/QuizFeature.swift`

