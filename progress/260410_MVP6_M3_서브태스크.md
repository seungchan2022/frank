# MVP6 M3 서브태스크 — 태그 탭 필터링 (tag_id 파라미터 + 독립 새로고침)

## 메인태스크 요약

탭 선택 시 해당 태그만 서버에 요청, 캐시 히트면 재요청 없이 즉시 표시.

- **서버**: `GET /me/feed?tag_id=<uuid>` Optional 쿼리 파라미터 (없으면 기존 join_all 유지)
- **웹**: `tagCache: Map<string, FeedItem[]>` 탭별 캐시 + 탭 전환 시 서버 재요청 (캐시 미스만)
- **iOS**: `tagCache: [String: [FeedItem]]` 탭별 캐시 + 탭 전환 시 서버 재요청 (캐시 미스만)

### 핵심 결정사항
- 캐시 히트 → 즉시 표시, 재요청 없음 (B 방식)
- Pull-to-refresh → 현재 탭 캐시 삭제 + 재요청
- 캐시 키: 웹 `'all' | uuid.string`, iOS `"all" | uuid.uuidString`
- 클라이언트 필터링(`filter()`, `articles` computed) 제거

**목적**: 구현 전 실패 테스트 작성

**작업**:
- `server/src/api/feed.rs` 테스트 섹션에 추가:
  - `get_feed_with_tag_id_returns_only_that_tag`: tag_id 쿼리 파라미터 있을 때 해당 태그 결과만 반환

  - `get_feed_without_tag_id_returns_all_tags`: tag_id 없으면 기존 join_all (하위 호환)

**산출물**: 컴파일 가능하고 `get_feed_with_tag_id_returns_only_that_tag`는 실패 상태

### S1: 서버 — tag_id 쿼리 파라미터 구현

**목적**: Optional tag_id로 단일 태그 검색 분기

**작업**:
- `server/src/api/feed.rs` 수정
  - `FeedQuery { tag_id: Option<Uuid> }` 구조체 추가 (`serde::Deserialize`)
  - `get_feed()` 파라미터에 `Query<FeedQuery>` 추가
  - `tag_id` 있을 때: 해당 태그 1개만 jobs Vec에 포함 → 기존 join_all 로직 그대로 통과
  - `tag_id` 없을 때: 기존 전체 태그 join_all (변경 없음)
  - `tag_id`가 사용자의 태그가 아닐 경우: 빈 배열 반환 (403 아님, 조용한 필터)
- 라우터에 `Query` 추출자 추가

**산출물**: S0 테스트 통과 + 기존 모든 테스트 유지

### S3: 웹 — API 클라이언트 + feedStore 탭 캐시 구현

**목적**: fetchFeed(tagId?) 시그니처 확장 + feedStore tagCache 로직

**작업**:
- `web/src/lib/api/client.ts`
  - `fetchFeed(tagId?: string): Promise<FeedItem[]>` 시그니처 변경
- `web/src/lib/api/realClient.ts`
  - `tagId` 있으면 `?tag_id=xxx` 쿼리 파라미터 추가
- `web/src/lib/api/mockClient.ts`
  - 시그니처 맞춤 (`tagId?: string` 추가, 동작은 기존 유지)
- `web/src/lib/stores/feedStore.svelte.ts`
  - `let tagCache = $state(new Map<string, FeedItem[]>())` 추가
  - `let activeTagId = $state<string | null>(null)` 추가
  - `selectTag(tagId: string | null)` async 함수 추가:
    - key = tagId ?? 'all'
    - 캐시 히트: `feedItems = tagCache.get(key)!; activeTagId = tagId; return`
    - 캐시 미스: fetch + 캐시 저장 + `feedItems = items; activeTagId = tagId`
    - isRefreshing/loading 중이면 no-op
  - `loadFeed()`: 초기 로드 시 `activeTagId = null`, 결과를 `tagCache`에 `'all'` 키로도 저장
  - `refresh()`: `tagCache = new Map([...tagCache].filter(([k]) => k !== (activeTagId ?? 'all')))` → 캐시 삭제 → 현재 탭 재요청 (activeTagId 기준)
  - `reset()`: `tagCache = new Map()`, `activeTagId = null` 추가
  - export에 `selectTag`, `activeTagId` 추가
- `npm run lint && npm run check && npm run test` 통과

**산출물**: S2 테스트 통과 + 기존 테스트 유지
