"읽기" 중심에서 "학습" 중심으로 전환한다.
사용자가 좋아요로 취향 시그널을 주면 피드가 개인화되고, 즐겨찾기한 기사로 퀴즈를 풀어 이해도를 확인한다.

## DB 스키마 변경 (M1에서 전부 확정)

### 신규 테이블: user_keyword_weights

```sql
CREATE TABLE user_keyword_weights (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id     UUID NOT NULL REFERENCES profiles(id) ON DELETE CASCADE,
    keyword     VARCHAR(100) NOT NULL,
    weight      INTEGER NOT NULL DEFAULT 1,
    updated_at  TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW(),
    UNIQUE(user_id, keyword)
);

CREATE INDEX idx_ukw_user_id ON user_keyword_weights(user_id);
```

- 좋아요 시 키워드 추출 → INSERT ... ON CONFLICT (user_id, keyword) DO UPDATE SET weight = weight + 1
- 개인화 threshold: SUM이 아닌 총 이벤트 횟수. `SELECT COUNT(DISTINCT updated_at)` 대신 별도 컬럼으로 관리 → like_count 컬럼 추가

```sql
-- like_count: 총 좋아요 이벤트 수 (threshold 판단용)
ALTER TABLE profiles ADD COLUMN like_count INTEGER NOT NULL DEFAULT 0;
```

### 기존 테이블 수정: favorites

```sql
ALTER TABLE favorites ADD COLUMN concepts JSONB;
```

concepts JSON 구조:
```json
[
  { "term": "용어명", "explanation": "한국어 설명" }
]
```

## M1: 버그픽스 + DB 스키마

### 목표
MVP6에서 이월된 iOS 타임아웃 버그를 제거하고, M2~M4에서 사용할 DB 스키마를 확정한다.
M1 완료 후 스키마는 변경하지 않는다.

### iOS 버그픽스

**대상**: iOS URLSession 타임아웃 미설정 → 요약 요청 30초 초과 시 실패

**실제 변경 파일**: `ios/Frank/Frank/Sources/Core/Adapters/APISummarizeAdapter.swift`

- `timeoutInterval` 35초 → **70초** (서버 60초 + 10초 여유)
- `URLSession.shared` 하드코딩 → **주입 가능한 구조** (`session: URLSession = .shared`)
- 기본 요청과 분리 완료

> 로드맵 원안(120초)에서 조정: 서버 OpenRouter timeout(60초)과 맞춰 70초로 확정.
> 실제 테스트에서 기존 30초 제한으로 실패하던 요약이 정상 동작 확인됨.

**추가 수정 (테스트 중 발견)**: `ArticleDetailFeature.swift`
- 즐겨찾기 기사 재진입 시 요약이 즉시 표시되지 않는 버그 수정
- `init()`에서 `SummarySessionCache` 히트 시 즉시 `phase = .done` 으로 시작

### 서버 타임아웃 수정 (추가)

**변경 파일**: `server/src/services/summary_service.rs`

- `SUMMARIZE_TIMEOUT_SECS` 30초 → **60초** (OpenRouter HTTP timeout과 일치)
- 기존엔 LLM 응답이 30~60초 구간에 오면 서버가 먼저 504 반환하는 구조적 버그

### DB 마이그레이션

**파일**: `supabase/migrations/20260411_mvp7_m1_schema.sql` — **Supabase 적용 완료**

1. `user_keyword_weights` 테이블 생성 (`weight CHECK (weight > 0)` 포함)
2. `profiles.like_count` 컬럼 추가
3. `favorites.concepts` 컬럼 추가 (`JSONB`)

### 서버 도메인 모델

**변경 파일**: `server/src/domain/models.rs`

- `UserKeywordWeight { user_id, keyword, weight: i32 }` 추가
- `QuizQuestion { question, options, answer_index: u8, explanation }` 추가 (`u8`: 보기 최대 4개)
- `QuizConcept { term, explanation }` 추가
- `Favorite.concepts: Option<serde_json::Value>` 필드 추가 (DB 컬럼 동기화)

## M2: 좋아요 + 키워드 추출

### 목표
피드 카드와 기사 상세에서 좋아요를 누르면 해당 기사의 키워드를 추출해 사용자 가중치 테이블에 누적한다.

### 서버

**작업 순서 (의존 순서대로)**

1. **LlmPort 확장** (`domain/ports.rs`)
   - `extract_keywords(title: &str, snippet: &str) -> Result<Vec<String>>` 추가
   - 기존 `summarize` 메서드 미수정

2. **OpenRouterAdapter 구현** (`infra/openrouter.rs`)
   - 기존 파일에 `extract_keywords` 구현 추가
   - 모델: `meta-llama/llama-3.3-70b-instruct:free` (OpenRouter 무료 tier)
   - 프롬프트: `"다음 기사 제목과 내용에서 핵심 기술 키워드 5개를 JSON 배열로만 반환해줘. 예: [\"iOS\", \"Swift\", \"SwiftUI\"]"`
   - max_tokens: 100 (키워드만이므로 짧음)

3. **DbPort 확장** (`domain/ports.rs`)
   - `increment_keyword_weights(user_id, keywords: Vec<String>) -> Result<()>`
   - `get_top_keywords(user_id, limit: usize) -> Result<Vec<String>>`
   - `increment_like_count(user_id) -> Result<i32>` (총 이벤트 수 반환)

4. **PostgresDbAdapter 구현** (`infra/postgres_db.rs`)
   - `increment_keyword_weights`: INSERT ... ON CONFLICT DO UPDATE weight + 1
   - `get_top_keywords`: ORDER BY weight DESC LIMIT N
   - `increment_like_count`: UPDATE profiles SET like_count = like_count + 1 RETURNING like_count

5. **POST /me/articles/like 엔드포인트** (`api/likes.rs` 신규)
   ```
   요청: { "title": "...", "snippet": "...", "url": "..." }
   처리: extract_keywords → increment_keyword_weights → increment_like_count
   응답: { "keywords": [...], "total_likes": N }
   ```

## M3: 피드 개인화 + 연관 기사

### 목표
누적된 키워드 가중치를 활용해 피드 검색 쿼리를 보강하고, 기사 상세에서 연관 기사를 제공한다.
피드 API 시그니처는 변경되지 않는다. 클라이언트는 연관 기사 UI만 추가한다.

### 서버

**작업 순서**

1. **피드 개인화 로직** (`services/feed_service.rs`)
   - `GET /me/feed` 내부에서 `get_top_keywords` 호출
   - `profiles.like_count >= 3` (PERSONALIZATION_THRESHOLD = 3) 이면:
     - 기존 태그 키워드 + 상위 3개 개인화 키워드를 AND 조합으로 검색 쿼리 보강
     - 예: `"iOS"` → `"iOS Swift mobile development"`
   - threshold 미달 시: 기존 동작 유지 (하위 호환)
   - API 시그니처 불변 (`GET /me/feed?tag_id=xxx` 동일)

2. **GET /me/articles/related 엔드포인트** (`api/related.rs` 신규)
   ```
   요청: GET /me/articles/related (쿼리 파라미터 없음 — 사용자 가중치 자동 참조)
   처리: get_top_keywords(user_id, 5) → SearchChainPort.search(keywords.join(" "), 5)
   응답: Vec<FeedItemResponse> (기존 피드 응답 구조 재사용)
   ```
   - 가중치 없으면 204 No Content 반환
   - 가중치 있으면 상위 5개 키워드로 검색 후 피드 형식 반환

## M4: 퀴즈

### 목표
즐겨찾기한 기사에서 핵심 개념 정리와 이해도 확인 퀴즈를 생성한다.
개념 정리는 favorites.concepts에 저장하고, 퀴즈 결과는 저장하지 않는다.

### 서버

**작업 순서**

1. **LlmPort 확장** (`domain/ports.rs`)
   - `generate_quiz(title: &str, content: &str) -> Result<QuizResult>` 추가
   - `QuizResult { concepts: Vec<QuizConcept>, questions: Vec<QuizQuestion> }` 반환

2. **OpenRouterAdapter 구현** (`infra/openrouter.rs`)
   - 기존 파일에 `generate_quiz` 구현 추가
   - 모델: `LLM_MODEL` 환경변수 (기존 설정 재사용 — 유료 모델 사용, 품질 중요)
   - 프롬프트:
     ```
     다음 기사에서 아래 두 가지를 JSON으로 반환해줘:
     1. "concepts": 핵심 기술 개념 3~5개. 각 항목은 { "term": "용어", "explanation": "한국어 설명 1~2문장" }
     2. "questions": 이해도 확인 객관식 문제 3개. 각 항목은 { "question": "질문", "options": ["A","B","C","D"], "answer_index": 0, "explanation": "해설" }
     ```
   - max_tokens: 1200
   - content 입력 우선순위: summary+insight (있으면) → snippet (없으면)

3. **FavoritesPort 확장** (`domain/ports.rs`)
   - `update_favorite_concepts(user_id, url, concepts: Vec<QuizConcept>) -> Result<()>` 추가

4. **PostgresFavoritesAdapter 구현** (`infra/postgres_favorites.rs`)
   - `UPDATE favorites SET concepts = $1 WHERE user_id = $2 AND url = $3`

5. **POST /me/favorites/{url}/quiz 엔드포인트** (`api/quiz.rs` 신규)
   ```
   요청: POST /me/favorites/{url}/quiz (바디 없음)
   처리:
     1. favorites에서 title + summary + insight + snippet 조회
     2. 즐겨찾기 미존재 시 404
     3. generate_quiz(title, summary ?? snippet)
     4. update_favorite_concepts(concepts)
     5. questions만 반환 (concepts는 저장만, 응답에 미포함)
   응답: { "questions": [...] }
   ```

### 웹 + iOS (서버 완료 후 동시 진행)

**웹**:
- `ArticleDetail` 상단 버튼 영역에 "퀴즈 풀기" 버튼 추가
  - 즐겨찾기한 기사만 버튼 표시
  - 클릭 시 `POST /api/favorites/{url}/quiz` 호출
