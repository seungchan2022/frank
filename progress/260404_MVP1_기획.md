## 요약

AI 기반 뉴스 스크랩 스터디앱의 MVP 1 기획.
키워드/태그 선택 → 자동 웹서치 → LLM 요약+인사이트 제공.

## 상세 내용

### 핵심 가치

AI 정보 과잉 문제를 해결 — 개인화 키워드 기반 자동 수집 + LLM 인사이트 제공.

### 아키텍처

```
Svelte 웹 (:5173) → SvelteKit 서버 프록시 → Rust API 서버 (:8080) → Supabase REST / 외부 API
```

- **Rust API 서버 (Axum :8080)**: 에코 서버 — 포트 호출 + 응답 변환만
- **Svelte 웹 프론트 (SvelteKit :5173)**: UI, API 서버로 프록시
- **Supabase**: DB + Auth (이메일 + Google/Apple 소셜), REST API로 접근
- **Python ML / iOS**: MVP 1 미사용

### 에코 서버 + 포트/어댑터 패턴

모든 앱(서버/웹/iOS)에 동일 패턴 적용:
- 앱은 에코 서버 — 포트 호출 + 응답 변환만
- 모든 외부 호출을 포트(trait/protocol)로 추상화
- State injection으로 프로덕션 어댑터와 Fake 어댑터 교체
- 포트는 관심사별 분리, 어댑터는 통일
- DB 접근은 Supabase REST API (sqlx 미사용)

### 기능 결정

| 항목 | 결정 |
|------|------|
| 핵심 기능 | 키워드/태그 선택 → 자동 웹서치 → LLM 요약+인사이트 |
| 플랫폼 | 웹 (Svelte) 먼저 |
| 서버 | Rust (Axum) API 서버 |
| 수집 소스 | Tavily → Exa → Firecrawl → arXiv (폴백 체인) |
| 본문 추출 | Firecrawl → Tavily Extract → Exa Contents → 스니펫 저장 |
| LLM | OpenRouter + Qwen 3.5 시리즈 |
| DB/Auth | Supabase (이메일 + Google/Apple 소셜) |
| DB 접근 | Supabase REST API (sqlx 미사용) |
| 웹→서버 통신 | SvelteKit 서버 프록시 |
| 화면 | 온보딩(태그) → 피드(매거진) → 상세(요약+인사이트) |
| 화면 확장 | 이후 스크랩 보관함 → 키워드 관리 순으로 추가 |
| 언어 | 다국어 수집 → 한국어 번역/요약 |
| 수집 트리거 | 사용자 새로고침 버튼 |
| UI 스타일 | 매거진 스타일 (썸네일+제목+요약) |
| 배포 | 로컬 Docker + Cloudflare 무료 |
