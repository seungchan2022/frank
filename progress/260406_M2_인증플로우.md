### Codex 리뷰 (기술적 타당성)

**핵심 발견 (5건):**

1. **signUp 계약 리스크 (Critical)** — Supabase 이메일 확인 활성 시 `signUp -> Profile` 계약 불성립. 반환형을 `Profile?` 또는 별도 `SignUpResult` enum으로 변경 권장
2. **ST-0 누락 (High)** — SupabaseClient 설정 주입(URL/Key), Apple Sign In capability/entitlement, Supabase 대시보드 Apple provider 설정이 빠져 있음. ST-6이 컴파일돼도 실 로그인 불가
3. **Apple 로그인 책임 분리 (Medium)** — `ASAuthorizationController`는 UI 프레젠테이션 컨텍스트가 필요하므로 Feature가 아닌 View/Coordinator에서 처리 후 `idToken/rawNonce`만 Feature에 전달해야 함
4. **파라미터명 rawNonce (Low)** — `nonce`보다 `rawNonce`가 명확 (Apple 요청에는 해시 nonce, Supabase에는 raw nonce 전달)
5. **Tuist 외부 패키지 추가 방식 (Medium)** — 현재 `Project.swift`에 `packages:` 파라미터 자체가 없음. `Package.swift` + `.external(name:)` 또는 `Project(packages:)` 방식 명시 필요

**Codex 권장 수정:**
- `signUp` 반환형 재검토 (`Profile?` 또는 `SignUpResult`)
- ST-0 추가: Supabase 설정 주입 + Apple capability + provider 설정
- `AuthFeature` 초기 상태를 `checkingSession`으로 시작
- Apple 로그인: `View/Coordinator -> Feature -> AuthPort` 흐름
- `Profile` 생성 규칙 명시: Auth user 매핑 vs profiles 테이블 조회/생성
