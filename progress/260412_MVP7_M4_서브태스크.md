## 메인태스크 요약

즐겨찾기한 기사에서 핵심 개념 정리와 이해도 확인 퀴즈를 생성한다.
개념 정리는 `favorites.concepts`에 저장하고, 퀴즈 결과는 저장하지 않는다.
M2·M3 코드는 수정하지 않는다.

---

## 인터뷰 확정 사항

| 항목 | 로드맵 원안 | 인터뷰 확정 | 이유 |
|------|-----------|-----------|------|
| URL 전달 방식 | `POST /me/favorites/{url}/quiz` (path param) | `POST /me/favorites/quiz` + body `{ "url": "..." }` | Axum path param에서 슬래시 포함 URL 처리 복잡, 기존 `/me/articles/like` 패턴과 일관성 |
| 퀴즈 형식 | 미명시 | 4지선다 객관식 | 구현 단순, LLM 구조화 용이 |
| 문제 수 | 미명시 | 3문제 | 기사 1개 분량에 적합, LLM 1회 호출로 생성 가능 |
| LLM 실패 처리 | 미명시 | 503 반환 (클라이언트 재시도 가능) | 능동적 요청 기능 — 사용자에게 재시도 기회 필요 |
| 개념 정리 캐시 | 미명시 | 항상 LLM 재호출 (캐시 없음) | MVP DoD 외 스코프 — 단순화 우선 |

## ST-1 상세: LlmPort.generate_quiz

### ports.rs 변경
```rust
async fn generate_quiz(
    &self,
    title: &str,
    content: &str,
) -> Result<QuizResult>;
```

`QuizResult`는 M1에서 이미 추가된 도메인 모델 사용:
```rust
// models.rs (M1 완료)
pub struct QuizResult {
    pub concepts: Vec<QuizConcept>,
    pub questions: Vec<QuizQuestion>,
}
pub struct QuizConcept { pub term: String, pub explanation: String }
pub struct QuizQuestion {
    pub question: String,
    pub options: Vec<String>,  // 4개
    pub answer_index: u8,
    pub explanation: String,
}
```

### OpenRouter 프롬프트
```
다음 기사에서 아래 두 가지를 JSON으로만 반환해줘 (다른 텍스트 없이):
1. "concepts": 핵심 기술 개념 3~5개. 각 항목은 { "term": "용어", "explanation": "한국어 설명 1~2문장" }
2. "questions": 이해도 확인 객관식 문제 3개. 각 항목은 { "question": "질문", "options": ["A","B","C","D"], "answer_index": 0, "explanation": "해설" }

제목: {title}
내용: {content}
```
- max_tokens: 1200
- 모델: `LLM_MODEL` 환경변수

### fake_llm.rs 변경
```rust
async fn generate_quiz(&self, _title: &str, _content: &str) -> Result<QuizResult> {
    Ok(QuizResult {
        concepts: vec![QuizConcept { term: "테스트 용어".into(), explanation: "테스트 설명".into() }],
        questions: vec![QuizQuestion {
            question: "테스트 질문?".into(),
            options: vec!["A".into(), "B".into(), "C".into(), "D".into()],
            answer_index: 0,
            explanation: "테스트 해설".into(),
        }],
    })
}
```

## ST-3 상세: POST /me/favorites/quiz

### 요청/응답
```
POST /me/favorites/quiz
Authorization: Bearer {token}
Content-Type: application/json

{ "url": "https://example.com/article" }

→ 200 OK
{ "questions": [
    {
      "question": "질문 텍스트",
      "options": ["A", "B", "C", "D"],
      "answer_index": 0,
      "explanation": "해설 텍스트"
    }
  ]
}

→ 404 Not Found (즐겨찾기 미존재)
→ 503 Service Unavailable (LLM 실패)
```

### 처리 흐름
1. `get_favorite_by_url(user_id, url)` → None이면 404
2. content = `summary + "\n" + insight` (있으면) 또는 `snippet`
3. `generate_quiz(title, content)` → 실패 시 503
4. `update_favorite_concepts(user_id, url, concepts)`
5. `questions`만 반환
