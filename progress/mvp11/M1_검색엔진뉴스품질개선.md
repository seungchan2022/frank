# M1: 검색엔진 뉴스 품질 개선 (BUG-004)

> 프로젝트: Frank MVP11
> 상태: 기획완료
> 예상 기간: 1~2일
> 의존성: 없음 (M2·M3·M4와 독립적)

## 목표

피드에 개별 기사 대신 뉴스 카테고리/태그 인덱스 페이지가 노출되는 문제를 수정한다.
각 검색엔진에 뉴스 전용 파라미터를 추가하고, URL 패턴 후처리 필터로 listing 페이지를 제거한다.

## 성공 기준 (Definition of Done)

- [ ] 피드에서 `/tag/`, `/category/`, `/topics/` 등 URL 패턴의 인덱스 페이지 0건 노출
- [ ] Exa 결과에 뉴스 기사 URL + `published_at` 포함 (날짜 필터 적용 확인)
- [ ] Tavily 결과에 `topic:"news"` 적용 후 listing 페이지 비율 감소 확인
- [ ] 기존 서버 테스트 전체 통과 (회귀 없음)
- [ ] 수동으로 피드 3회 새로고침 — 인덱스 페이지 0건 확인

## 폴백 체인 (실제)

`server/src/main.rs` 기준: **Tavily → Exa → Firecrawl** (3개)

## 엔진별 현황 및 수정 방향

### Exa (`infra/exa.rs`) — 최우선

현재 파라미터:
```json
{
  "query": "...",
  "numResults": 5,
  "contents": { "highlights": { "numSentences": 3, "highlightsPerUrl": 1 } }
}
```

추가할 파라미터:
```json
{
  "type": "news",
  "startPublishedDate": "<현재날짜 - 7일 ISO8601>"
}
```

> Exa가 현재 메인 엔진 (Tavily rate limit 초과 시 자동 폴백됨). 수정 효과 즉시 반영.

### Tavily (`infra/tavily.rs`)

현재 파라미터:
```json
{
  "query": "...",
  "max_results": 5,
  "search_depth": "advanced",
  "include_answer": false,
  "time_range": "week"
}
```

추가할 파라미터:
```json
{
  "topic": "news"
}
```

> `topic:"news"` + `time_range:"week"` 조합 동작 여부를 실제 API 응답으로 검증 필요.
> 현재 플랜 rate limit(432) 상태 — 한도 리셋 후 동작 확인.

### Firecrawl (`infra/firecrawl.rs`) — 후처리로만 보완

- `/v1/search` API에 뉴스·날짜 필터 파라미터 자체 없음
- `published_at`, `image_url` 미반환
- URL 패턴 필터(공통 후처리)로만 listing 페이지 제거 가능

### 공통 후처리 필터 (`infra/search_chain.rs`)

listing URL 패턴 제외:
- `/tag/`, `/tags/`
- `/category/`, `/categories/`
- `/topics/`, `/topic/`
- `/section/`

## 리스크

| 리스크 | 영향(H/M/L) | 대응 |
|--------|------------|------|
| Exa `type:"news"` 적용 시 결과 수 감소 | M | `numResults` 증가 또는 fallback 조정 |
| Tavily `topic + time_range` 조합 미지원 | M | `time_range` 제거 후 `topic`만 적용 테스트 |
| URL 필터로 유효한 기사 제외 | L | 필터 패턴을 경로 단독 매칭으로 제한 (도메인 제외) |
