# Frank MVP11 로드맵

> 생성일: 260423
> 상태: in-progress
> 원칙: 버그 수정 우선 → 서버 먼저 → 웹·iOS 독립 병렬 진행

## 목표

실기기/시뮬레이터 테스트에서 발견된 버그 4개를 수정하고, 뉴스 수집 품질을 개선한다.
새 기능 추가 전 앱 기본 신뢰성과 콘텐츠 품질을 확보하는 것이 이번 MVP의 전부다.

## 버그 현황

| ID | 위치 | 심각도 | 마일스톤 |
|----|------|--------|---------|
| BUG-001 | iOS | 높음 | M2 |
| BUG-002 | iOS | 높음 | M2 |
| BUG-003 | iOS+웹 | 중간 | M3(웹), M4(iOS) |
| BUG-004 | 서버 | 중간 | M1 |

## 마일스톤 구성

```
M1 (서버) ──────────────────────────────────────────▶ done
           │
           └─ M2 (iOS 초기화·환경설정) ──────────────▶ done
           └─ M3 (웹 즐겨찾기·오답 태그 필터) ────────▶ done   ← 병렬
           └─ M4 (iOS 즐겨찾기·오답 태그 필터) ───────▶ done   ← 병렬
```

> M1 완료 후 M2·M3·M4는 서로 독립적으로 병렬 진행 가능.
> 서버 API는 BUG-003 관련 태그 데이터를 이미 반환 중 → 서버 추가 변경 불필요.

|---|---|---|---|---|---|
| M1 | 검색엔진 뉴스 품질 개선 | BUG-004 | 서버 | 기획완료 | 1~2일 |
| M2 | iOS 초기화·환경설정 버그 수정 | BUG-001, BUG-002 | iOS | in-progress | 1~2일 |
