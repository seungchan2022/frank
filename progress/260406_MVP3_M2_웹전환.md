# M2: 웹 전환

> 상태: 📋 기획 완료
> 범위: 서버 + 웹
> 의존: M1

## 목표

웹의 Supabase 직접 호출 15개를 Rust API 호출로 전환 + localStorage → httpOnly 쿠키 세션 전환.

## 전환 대상 (api.ts의 Supabase 호출)

| 현재 함수 | Supabase 호출 | 전환 대상 Rust API | 상태 |
|----------|-------------|-------------------|------|
| fetchTags() | `from('tags').select()` | GET /api/tags | ✅ 준비됨 |
| fetchMyTagIds() | `from('user_tags').select()` | GET /api/me/tags | ✅ 준비됨 |
| saveMyTags() | `delete` + `insert` + `profiles.update` | POST /api/me/tags | ✅ 준비됨 |
| updateMyTags() | `delete` + `insert` | POST /api/me/tags | ✅ 준비됨 |
| fetchProfile() | `from('profiles').select().single()` | GET /api/me/profile | ✅ 준비됨 |
| fetchArticles() | `from('articles').select().order().range()` | GET /api/me/articles?limit=&offset=&tag_id= | M1에서 확장 |
| fetchArticleById() | `from('articles').select().eq().single()` | GET /api/me/articles/:id | M1에서 추가 |
| collectArticles() | SvelteKit 프록시 → Rust | POST /api/me/collect | ✅ 직접 호출로 변경 |
| summarizeArticles() | SvelteKit 프록시 → Rust | POST /api/me/summarize | ✅ 직접 호출로 변경 |
| getAuthHeaders() | `supabase.auth.getSession()` | SDK 유지 (토큰 추출만) | ⚠️ |

## 서브태스크

### ST-1: @supabase/ssr 설치 + 쿠키 기반 세션 전환
- **유형**: feature
- **상태**: [ ]
- **설명**:
  1. `npm install @supabase/ssr` 
  2. `web/src/hooks.server.ts` 생성 — createServerClient + 쿠키 어댑터
  3. `web/src/lib/supabase.ts` → 인증 전용으로 축소
  4. localStorage 세션 → httpOnly 쿠키로 전환
- **산출물**: `hooks.server.ts`, `supabase.ts` 수정
- **의존**: 없음

### ST-2: api.ts 전면 재작성 (Supabase → fetch)
- **유형**: feature
- **상태**: [ ]
- **설명**: 모든 데이터 호출을 `fetch(Rust API URL)` + Bearer 토큰으로 변경
- **삭제 대상**: Supabase `.from().select()` 호출 전부
- **유지 대상**: `supabase.auth.getSession()` (토큰 추출용)
- **산출물**: `api.ts` 재작성
- **의존**: M1 완료 (articles 확장, articles/:id 추가)

### ST-3: SvelteKit API 프록시 제거
- **유형**: chore
- **상태**: [ ]
- **설명**: `web/src/routes/api/collect/+server.ts`, `web/src/routes/api/summarize/+server.ts` 제거. 클라이언트가 Rust 서버 직접 호출.
- **주의**: CORS 설정 필요 (Rust 서버에서 웹 origin 허용)
- **산출물**: 프록시 파일 삭제, Rust 서버 CORS 미들웨어 추가
- **의존**: ST-2

### ST-4: 페이지별 동작 검증
- **유형**: test
- **상태**: [ ]
- **설명**: 각 페이지(login, onboarding, feed, feed/[id], settings)가 Rust API로 정상 동작하는지 확인
- **산출물**: 기존 테스트 수정 + Playwright 시각적 검증
- **의존**: ST-1, ST-2, ST-3

### ST-5: 웹 테스트 갱신
- **유형**: test

- **의존**: ST-2
