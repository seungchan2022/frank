# M2: 웹 전환

> 상태: 📋 기획 완료 (M1.5 후 병렬 시작)
> 범위: **웹만** (server/와 ios/는 read-only)
> 의존: M1 ✅, M1.5 ✅
> 작업 탭: `frank-m2-web` (worktree)
> 브랜치: `feature/260408_m2_web`

## 핵심 원칙 (절대 위반 금지)

1. **격리**: web/ 디렉토리만 수정. server/와 ios/는 절대 건드리지 말 것
2. **Mock-First**: ST-3까지는 외부 자원(Supabase, Rust 서버) 호출 0건 유지
3. **M1.5 인프라 강제 사용**: `web/src/lib/api/`의 `ApiClient`/`MockApiClient`/`RealApiClient`를 우회하지 말 것. 옛 `api.ts` 직접 호출 방식으로 회귀 금지
4. **공통 문서 손대지 말 것**: `progress/` 폴더에서 본 문서(`260406_MVP3_M2_웹전환.md`)와 회고만 수정. 로드맵, M3 문서, M1.5 문서는 read-only
5. **Server 작업 발견 시 즉시 정지**: ST-5에서 CORS 등 server/ 변경이 필요해 보이면 작업 멈추고 사용자에게 보고. 메인 탭에서 hotfix 처리

## 작업 흐름

```
1. Mock 모드로 모든 페이지 동작 확인 (외부 호출 0)
   ↓
2. 컴포넌트가 $lib/api에서 직접 import하도록 마이그레이션 (utils/api.ts shim 의존 제거)
   ↓
3. RealApiClient 내부를 Supabase 직접 호출 → fetch(Rust API)로 교체
   (인터페이스/컴포넌트 무변경)
   ↓
4. @supabase/ssr 쿠키 세션 전환
   ↓
5. SvelteKit API 프록시 (/api/collect, /api/summarize) 제거
   ↓
6. 테스트 갱신 + Playwright E2E
   ↓
7. 머지 준비 + main rebase
```

**M3(iOS)와 완전 독립**. 두 worktree 탭에서 동시 작업 시 외부 자원 공유 없음.

## 목표

웹의 모든 데이터 호출을 Rust API로 통합 + httpOnly 쿠키 세션 전환. M1.5에서 만든 `ApiClient` 추상화 위에서 진행하므로 컴포넌트 코드는 최소 변경.

## 사실 확인 (M1.5 이후 현재 상태)

- ✅ `web/src/lib/api/` 인프라 존재 (`client.ts`, `mockClient.ts`, `realClient.ts`, `types.ts`, `index.ts`, `__fixtures__/`)
- ✅ `web/src/lib/utils/api.ts`는 `apiClient`로 위임하는 shim
- ✅ 5개 페이지가 `$lib/utils/api`에서 import 중 (`feed`, `feed/[id]`, `settings`, `onboarding`, `+page.svelte`)
- ✅ `web/src/lib/utils/api.test.ts` 존재 (390줄, Supabase mock 기반)
- ✅ SvelteKit 프록시 2개 존재 (`web/src/routes/api/collect`, `web/src/routes/api/summarize`)
- ✅ `VITE_USE_MOCK_API=true` 환경변수로 Mock/Real 스위치 가능

## 서브태스크 (Mock-First 순서)

### ST-1: Mock 모드 전체 페이지 동작 검증
- **유형**: test/feature
- **상태**: [ ]
- **목표**: `VITE_USE_MOCK_API=true`로 dev 서버 띄우고 5개 페이지(login, onboarding, feed, feed/[id], settings) 모두 동작 확인. **외부 호출 0건**
- **방법**:
  1. `web/.env.local`에 `VITE_USE_MOCK_API=true` 추가 (Git 추적 안 됨)
  2. `npm run dev`
  3. 각 페이지를 Playwright/Chrome DevTools로 스크린샷
  4. 네트워크 탭에서 supabase.co/rust 서버 호출 0건 확인
- **이슈 발견 시**: MockApiClient의 메서드 시그니처/응답이 컴포넌트 기대와 다르면 `mockClient.ts`만 수정 (api.ts/페이지 코드는 그대로)
- **수정 파일**: `web/.env.local` (gitignore), 필요 시 `web/src/lib/api/mockClient.ts`
- **의존**: 없음

### ST-2: 컴포넌트 import 마이그레이션 ($lib/utils/api → $lib/api)
- **유형**: refactor
- **상태**: [ ]
- **목표**: 5개 페이지가 shim(`$lib/utils/api`) 대신 `$lib/api`에서 직접 `apiClient`를 import하도록 정리
- **방법**:
  - 각 페이지에서 `import { fetchArticles, ... } from '$lib/utils/api'` 제거
  - `import { apiClient } from '$lib/api'`로 교체
  - 호출부를 `await apiClient.fetchArticles({offset, limit, tagId})` 형태로 수정
- **유지**: `getAuthHeaders()`는 `$lib/utils/api`에 남겨도 됨 (인증 토큰 추출)
- **수정 파일**: 5개 페이지 + (선택) `web/src/lib/utils/api.ts` 정리
- **의존**: ST-1 (Mock 동작 확인 후 진행해야 안전)
- **검증**: ST-1과 동일하게 Mock 모드에서 모든 페이지 동작 확인 (회귀 0)

### ST-3: RealApiClient를 Rust API로 전환
- **유형**: feature
- **상태**: [ ]
- **목표**: `web/src/lib/api/realClient.ts`의 메서드 내부를 Supabase 직접 호출 → `fetch(VITE_RUST_API_URL/...)` + Bearer 토큰으로 교체
- **인터페이스 무변경**: `ApiClient` 시그니처는 그대로. 컴포넌트 수정 0
- **유지**: `supabase.auth.getSession()` (토큰 추출 전용)
- **환경변수**: `VITE_RUST_API_URL` 추가 (예: `http://localhost:8080`)
- **수정 파일**: `web/src/lib/api/realClient.ts`, `web/.env.local`
- **참조**: `progress/260407_API_SPEC.md`의 엔드포인트 표
- **의존**: ST-2
- **검증**: `VITE_USE_MOCK_API=false`로 dev 서버 실행 + Rust 서버 실행 후 5개 페이지 동작 확인
