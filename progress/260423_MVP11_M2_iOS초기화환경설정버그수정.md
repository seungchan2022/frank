## 현재 상태 파악 (코드 분석)

### BUG-001: 세션 초기화 순서 문제

**흐름 분석**:
```
FrankApp.init()
  → AppDependencies.live()
      → SupabaseClient(supabaseURL:, supabaseKey:)  ← 옵션 없이 생성
  → AuthFeature(auth: deps.auth)

RootView.body
  case .checkingSession → SplashView(feature: feature)
    .task { await feature.send(.checkSession) }  ← 비동기 세션 복원 시작
      → client.auth.session  ← Supabase가 로컬 세션 복원 (비동기)
```

**문제**: `SupabaseClient` 초기화 시 `emitLocalSessionAsInitialSession` 옵션이 없어,
세션 복원이 비동기로 늦게 일어남. 콘솔에서도 경고 확인됨:
```
Initial session emitted after attempting to refresh the local stored session.
To opt-in to the new behavior now, set `emitLocalSessionAsInitialSession: true`
```

**수정 대상 파일**: `ios/Frank/Frank/Sources/App/AppDependencies.swift`
- `SupabaseClient` 생성 코드에 `AuthClientConfiguration` 옵션 추가

**수정 방향 — 옵션 A (권장)**:
```swift
let client = SupabaseClient(
    supabaseURL: config.url,
    supabaseKey: config.anonKey,
    options: SupabaseClientOptions(
        auth: .init(storage: .keychain, flowType: .pkce, autoRefreshToken: true),
        // emitLocalSessionAsInitialSession은 AuthClientConfiguration에 있음
    )
)
```
→ context7 MCP로 Supabase iOS SDK 최신 API 확인 후 정확한 옵션명 적용 필요.

**수정 방향 — 옵션 B (폴백)**:
`SplashView`/`RootView`에서 `.checkingSession` 상태 유지 중에는
하위 API 요청 뷰(`MainTabView`)가 절대 생성되지 않으므로 현재 구조상 BUG-001의
원인은 RootView 분기보다 세션 복원 전 다른 경로 호출 가능성.
→ 옵션 A 적용 후 재현 여부 확인.

### BUG-002: ServerConfig 시뮬레이터/실기기 URL 분기 없음

**현재 코드** (`ServerConfig.swift`):
```swift
static var live: ServerConfig {
    // 1) Info.plist → SERVER_URL
    // 2) Secrets.plist → SERVER_URL
    // 3) localhost:8080 폴백
}
```

**문제**: `Config.xcconfig`에 `SERVER_URL = http://localhost:8080`이 설정되어 있지만,
실기기 테스트 시 LAN IP로 바꾸면 시뮬레이터에서 연결 실패. 컴파일 타임 분기 없음.

**수정 대상 파일**: `ios/Frank/Frank/Sources/Core/Config/ServerConfig.swift`
- `#if targetEnvironment(simulator)` 블록으로 시뮬레이터는 `localhost:8080` 하드코딩
- 실기기는 기존 Info.plist → Secrets.plist → localhost 폴백 순서 유지
