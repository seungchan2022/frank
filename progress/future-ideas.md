## 앱 정체성 재정의 (2026-04-29 대화에서)

기존: "AI 기반 뉴스 스크랩 스터디앱"
→ 수정: **"내 관점으로 트렌드를 보는 개인 레이더"**

퀴즈/오답노트는 부가 기능. 핵심은 **내 배경(예: iOS 개발자)으로 뉴스를 읽고, 관련 인사이트를 쌓아가는 것**.

---

## 기능 아이디어

### [IDEA-01] 내 관점 기반 AI 인사이트
요약 시 단순 요약이 아닌 사용자 배경(iOS 개발자 등) 기반 맞춤형 인사이트 생성.
예: "iOS 개발자 시각에서 이 기사의 핵심은 X이고, 실무에서 Y로 연결됩니다"
- 구현: 요약 프롬프트에 사용자 프로필/역할 컨텍스트 추가
- 서버 변경 최소 (프롬프트 개선 수준)

### [IDEA-02] 기사에 내 생각 메모
기사를 읽고 나서 나만의 코멘트를 남기는 기능.
나중에 "이 트렌드에 대해 내가 어떻게 생각했었나"를 시간순으로 돌아볼 수 있음.
- DB: `article_memos` 테이블 (user_id, article_url, memo, created_at)
- 웹+iOS 양쪽 지원

### [IDEA-03] 트렌드 키워드 추적
내 관심 태그에서 반복적으로 등장하는 키워드 시각화.
예: "이번 달 AI 탭에서 'Agent'가 12번 등장"
- 서버에서 키워드 빈도 집계 필요

### [IDEA-04] 읽기 스트릭 / 학습 통계
"오늘 3개 읽음 · 5일 연속" 같은 간단한 지표.
꾸준히 앱을 열게 만드는 동기부여. 퀴즈 배지 시스템과 자연스럽게 연결 가능.

### [IDEA-05] 알림 (클라우드 배포 후)
- M3 클라우드 배포 전: 의미 없음 (서버가 맥 로컬에만 있어서)
- M3 이후: 로컬 알림(Background App Refresh) 또는 APNs 푸시 알림
- 트리거: 새 기사 도착, 복습 스케줄 등

## 우선순위 판단 기준

- **앱 정체성 부합**: "내 관점 + 트렌드 레이더"에 맞는가
- **서버 부하**: 현재 로컬 서버 기준 감당 가능한가
- **클라우드 의존**: M3 배포 전/후 중 언제 가능한가
