## 분해 원칙

- **단일 책임**: 각 서브태스크는 하나의 목적·하나의 산출물.
- **순수 로직 ↔ I/O 셸 분리**: Rust 포트/어댑터 패턴 유지. 순수 로직(변형 생성·번역 매핑·실패 분류·포맷터)은 표 기반 단위 테스트, I/O(stdin·HTTP·파일·DB)는 셸로 격리.
- **독립 실행**: S1 도착 후 S2/S3/S5는 병렬 가능. S4는 S1·S2 완료 후 시작.
- **적정 크기**: 1~2시간 내 완료. 90% 커버리지 유지(rust-scope).
