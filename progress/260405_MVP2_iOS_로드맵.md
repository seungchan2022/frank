## 아키텍처

### 핵심 패턴: Feature-Driven Unidirectional Flow

```
View(탭) → 클로저 → Feature.send(Action) → Reducer(Mutation) → 프로퍼티 갱신 → 해당 서브뷰만 리렌더
                         ↓
                    Port(Protocol) → Adapter (API 호출)
                         ↓
                    도메인 모델 → Component State 직렬화
```

### 레이어 역할

| 레이어 | 역할 | 상태 소유 | 테스트 |
|--------|------|----------|--------|
| Feature (@Observable) | Action 수신 + API 호출(Port) + Reducer + 프로퍼티별 상태 관리 | O (유일) | Mock Port 주입 → Action → 프로퍼티 검증 |
| Component State | 컴포넌트 단위 상태 (struct, 읽기 전용) | X | 순수 함수 빌드 테스트 |
| View | 서브뷰 분리 + 클로저로 Action dispatch | X | 로직 없음 |
| Component | Props(State) + 클로저 → UI 렌더링 | X | 스냅샷/렌더링 테스트 |

### @Observable 세분화 렌더링 규칙

`@Observable`은 프로퍼티 단위 추적이지만, body 단위로 적용된다.
세분화된 리렌더를 위해 다음 규칙을 따른다:

1. Feature의 상태를 **프로퍼티별로 분리** (단일 ViewState struct 금지)
2. 부모 View에서 **Feature 프로퍼티를 직접 읽지 않음**
3. **@Observable 객체(Feature) 자체를 서브뷰에 전달**
4. 서브뷰 body에서 **필요한 프로퍼티만 읽음**

```swift
// Feature: 프로퍼티별 분리
@Observable class FeedFeature {
    var tagBar: TagChipBarState       // 독립 추적
    var cards: [ArticleCardState]     // 독립 추적
    var shimmer: ShimmerState         // 독립 추적
}

// 부모 View: 프로퍼티 읽지 않음, feature 객체만 전달
struct FeedView: View {
    var feature: FeedFeature
    var body: some View {
        TagSection(feature: feature)    // tagBar 변경 시에만 리렌더
        CardSection(feature: feature)   // cards 변경 시에만 리렌더
    }
}

// 서브뷰: 필요한 프로퍼티만 읽음
struct TagSection: View {
    var feature: FeedFeature
    var body: some View {
        TagChipBarView(state: feature.tagBar) { tagId in
            Task { await feature.send(.selectTag(tagId)) }
        }
    }
}
```

### 의존 방향

```
View → Feature(@Observable) → Ports(Protocol) ← Adapters(URLSession/Supabase)
                 ↓
          Component State (struct)
```

## API 엔드포인트 (iOS에서 호출)

### Supabase REST (직접 호출)

| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | `/api/tags` | 전체 태그 목록 |
| GET | `/api/me/tags` | 내 구독 태그 ID 목록 |
| POST | `/api/me/tags` | 태그 저장 `{tag_ids: [UUID]}` |
| GET | `/api/me/profile` | 프로필 (온보딩 완료 여부) |
| GET | `/api/me/articles?limit=N` | 내 기사 목록 |

### Rust 서버 (Bearer 토큰 전달)

| 메서드 | 경로 | 설명 |
|--------|------|------|
| POST | `/api/me/collect` | 기사 수집 트리거 |
| POST | `/api/me/summarize` | AI 요약 트리거 |

### 인증

Supabase Auth (이메일/패스워드) → JWT Bearer 토큰
