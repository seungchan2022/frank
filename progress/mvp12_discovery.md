## 1. 버그 (수정 필수)

### BUG-A: BBC 스타일 토픽 URL 필터 누락
- **플랫폼**: 서버 (피드 공통)
- **증상**: `bbc.com/news/topics/cz4pr2gd85qt` 같은 토픽 인덱스 페이지가 피드에 기사로 노출됨
- **원인**: `is_listing_url()` 필터에 `/news/topics/` 패턴 누락
- **수정 방향**: `feed.rs` URL 패턴 필터에 `/topics/` 패턴 추가

---

### BUG-B: 기사 소개(snippet) 마크다운·네비게이션 텍스트 오염
- **플랫폼**: 서버 (피드 공통)
- **증상**: snippet에 `###`, `####` 마크다운 헤더, 댓글 수, 저자, 목차 등 본문 외 텍스트 포함
- **예시**:
  - `"이젠 말로 디자인한다 [...] ### 제목 [...] #### 소제목 [...]"` — 마크다운 헤더 노출
  - `"Figma 떨고 있나? [...] 댓글 0 · 저장 0 · 편집: 이도윤 기자 [...]"` — 메타 텍스트 노출
- **원인**: `clean_snippet()`이 HTML 태그는 제거하지만 마크다운 문법·네비게이션 텍스트는 미처리
- **수정 방향**: `clean_snippet()` 보강 — `#` 헤더 제거, `[...]` 생략 패턴 정리, 메타 텍스트 필터

---

### BUG-C: 웹 스크랩 — orphan selectedTagId (태그 사라진 후 선택 없음)
- **플랫폼**: 웹
- **재현**:
  1. 스크랩 탭 → 특정 태그(예: 투자) 선택
  2. 해당 태그의 마지막 기사 즐겨찾기 해제
  3. "해당 태그의 즐겨찾기 기사가 없습니다." 표시 후 태그 칩 사라짐
  4. 아무 태그도 선택되지 않은 상태 → 전체 목록도 안 보임
- **수정 방향**: 태그 삭제 시 `selectedTagId`를 `null`(전체)로 자동 초기화

---

### BUG-D: iOS 즐겨찾기 추가 시 새 태그 미생성
- **플랫폼**: iOS
- **재현**:
  1. 스크랩 탭에 "모바일 개발" 태그만 있는 상태
  2. 피드에서 AI 관련 기사 즐겨찾기 추가
  3. 스크랩 탭 확인 → "AI" 태그가 생기지 않음
- **원인 추정**: 즐겨찾기 추가 후 태그 목록 리프레시 미호출
- **수정 방향**: 즐겨찾기 추가 이벤트 시 스크랩 탭 태그 목록 재조회

---

### BUG-E: iOS 스크랩 — 마지막 기사 삭제 후 아무 반응 없음
- **플랫폼**: iOS
- **재현**:
  1. 스크랩 탭에서 태그 선택
  2. 해당 태그의 마지막 기사 즐겨찾기 해제
  3. 태그 칩은 선택된 채로 남고 화면에 아무 텍스트도 표시 안됨 (빈 화면)
- **수정 방향**: 웹 BUG-C와 동일 — 태그 삭제 시 `selectedTagId` nil 초기화 + "기사 없음" 메시지 표시

---

### BUG-F: 웹 + iOS 오답노트 필터링 작동 안함
- **플랫폼**: 웹 + iOS
- **증상**:
  - 오답노트 탭에 스크랩 탭과 동일한 태그 필터가 표시됨
  - 태그 칩을 눌러도 필터링이 실제로 동작하지 않음
- **원인 추정**: 오답노트 태그 필터가 스크랩의 상태를 공유하거나, 필터 핸들러가 오답노트 데이터에 연결 안됨
- **수정 방향**: 오답노트 전용 태그 필터 상태 분리 + 필터 핸들러 연결

## 3-1. 기능 개선 — 좋아요 UX 재설계

### 현재 문제
- 피드 카드에 좋아요(♡)와 즐겨찾기(★)가 나란히 배치 → 사용자가 차이를 인식하기 어려움
- 피드에서는 즐겨찾기 기능을 실제로 사용할 수 없는데 버튼이 존재
- 좋아요의 목적(키워드 추출 → 피드 개인화)이 사용자 눈에 보이지 않음

### 결정
- 좋아요 기능을 즐겨찾기에 통합 (즐겨찾기 시 키워드 추출도 같이 처리)
- 별도 좋아요 버튼 제거
- 구체적인 UX 배치는 마일스톤에서 결정
