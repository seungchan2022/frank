최우선 1개만 권장

🥇
G1. Feature List 자동 생성
무엇
서브태스크 시작 시 AI가 "완료되려면 충족해야 할 세부 조건"을 자동 생성해 체크리스트로 남김
지금 부족
현재 완료 조건은 포괄적 항목 몇 개만 (린트·테스트·리뷰 통과). 태스크별 엣지 케이스·에러 처리·회귀 검증이 누락 → AI가 "다 했어요" 선언해도 검증 불가
어떻게
서브태스크 인터뷰 직후 태스크별 체크리스트 자동 생성 단계 추가
효과
Anthropic 실패 유형 중 대충 끝내기·거짓 완료 선언·방향 이탈 동시 대응. 난이도 낮음 / 반나절 작업

필요해질 때 꺼내 쓸 항목

나머지 3개 Gap

G6
세션 MCP 과다 로드
무엇
각 스킬이 이번 작업에 쓸 도구만 미리 지정 → 전역 200개 중 일부만 로드
지금 부족
22개 스킬 중 도구 지정 없는 3개는 전역 200개 전부 로드. 일부 스킬은 도구 수 과다(17~25개)
어떻게
지정 없는 스킬부터 추가 → 과다 스킬 다이어트 → 중간 스킬 점진 조정. 설정 파일 편집만
언제
토큰 비용 체감 시, 도구 선택 오류 잦을 때. 난이도 매우 낮음
G3
UI 자율 피드백
무엇
AI가 자기가 만든 화면을 직접 보고, 스스로 수정 반복. "구현 → 스크린샷 → 기준과 비교 → 불일치 시 재수정" 루프
지금 부족
UI 변경 시 사용자가 매번 수동 확인. 반복 수정 시 사이클마다 사람 개입 필요
어떻게
검증 단계에 스크린샷 촬영 + 기준 비교 + 불일치 시 구현 단계 재진입 루프 추가
언제
전제 조건 2가지: ① 디자인 시안/회귀 기준 있음 ② 같은 UI 3번↑ 반복 수정 중. 토큰 비용 큼
G4
MVP 완료 아카이빙 알림
무엇
로드맵 수정을 훅이 감지 → 모든 마일스톤이 "완료"면 AI가 자동으로 아카이빙 제안 → 사용자 승인 시 실행
지금 부족
아카이빙 스킬은 있으나 사용자가 "끝났지" 기억해서 수동 실행해야 함. 잊으면 낡은 파일 계속 쌓임
어떻게
훅이 로드맵 변경 감지 → AI에 제안 주입
언제
MVP 여러 개 돌리면서 아카이빙을 자주 잊는 시점
