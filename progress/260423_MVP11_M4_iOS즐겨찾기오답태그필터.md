# MVP11 M4 메인태스크: iOS 즐겨찾기·오답 태그 필터

## 분석 요약

### 재사용 가능한 컴포넌트

| 컴포넌트 | 경로 | 비고 |
|---|---|---|
| `TagChipBarView` | `Components/TagChipBar/TagChipBarView.swift` | `tags: [Tag]`, `selectedTagId: UUID?`, `onSelect: (UUID?) -> Void` 인터페이스 |
| `TagChipView` | `Components/TagChipView.swift` | 개별 칩 + `tagChipStyle()` ViewModifier |

`TagChipBarView`는 `UUID?` 기반으로 태그를 선택한다. `FavoritesFeature`의 `items: [FavoriteItem]`은 `tagId: UUID?`를 이미 보유하고 있어 **그대로 재사용 가능**하다.

### FavoriteItem 태그 구조

- `FavoriteItem.tagId: UUID?` — 서버에서 저장·반환 중
- 필터링: `tagId`로 직접 UUID 비교 → `TagChipBarView`와 타입 일치
- 태그 이름 복원: `tagId → Tag.name`은 `MockFixtures.tags`에 있으나, 실제 앱에서는 `TagPort.fetchAllTags()`가 필요
  - **결론**: 즐겨찾기 목록의 `tagId`들만 추출 → `AppDependencies`가 주입하는 `TagPort`에서 이름을 조회하는 대신, **`FavoritesView` 초기화 시 `tag` 포트를 추가 주입하거나**, `items`의 `tagId`만으로 UUID 필터를 적용하고 칩 이름은 "태그 없음" fallback으로 처리하는 방식 중 하나를 선택해야 한다.
  - **채택 방향**: 기획 문서 목표 범위(클라이언트 사이드 필터)에 따라 `FavoritesFeature`에 `availableTags: [Tag]` computed property를 추가하고, 태그 이름은 별도 `TagPort` 호출 없이 items에서 태그 ID만 수집한 뒤 `Tag(id:, name: "태그ID", category: nil)` 대신 **Feature가 `[Tag]` 목록을 외부에서 주입받는 방식**으로 구현. 단, `FavoritesView`는 이미 `TagPort`를 갖고 있지 않으므로 `TagPort`를 별도 주입하거나, items에서 tagId와 이름을 pair로 추출하는 헬퍼를 사용하는 방안 중 후자가 범위를 최소화함.
  - **최종 결정**: `FavoritesFeature`에 `availableTags: [Tag]` computed property 추가 — `items`의 `tagId`로 UUID를 수집하고 이름은 `"(tagId 앞 8자)"` 대신, `AppDependencies`에서 주입할 `allTags: [Tag]`를 `FavoritesFeature`에 저장해 이름 매핑 → `FavoritesView` 초기화 시 `allTags`를 주입하는 구조. **단, 범위 최소화를 위해** `allTags` 주입 없이 items의 tagId를 기반으로 UUID 필터만 구현하고, 칩 이름은 tagId를 Tag(id:name:category:)로 감싸되 이름은 외부에서 넘어온 `[Tag]` 목록과 매칭한다. 이를 위해 `AppDependencies`→`FavoritesView` 경로에 `allTags: [Tag]` 파라미터를 추가한다.

### WrongAnswer 태그 구조 (핵심 제약)

- `WrongAnswer` 모델에 `tagId` 필드 **없음** (DB 스키마에도 없음)
- 오답과 태그를 연결하는 유일한 키: `articleUrl: String`
- `FavoriteItem`에는 `url`과 `tagId` 모두 있음
- **필터 전략**: `WrongAnswersFeature`가 `FavoritesFeature`의 `items`를 참조하거나, `WrongAnswer.articleUrl`로 `FavoriteItem` 목록을 조회해 tagId를 추론 → 오답이 즐겨찾기에 없는 기사일 수도 있으므로 매칭 실패 시 "태그 없음" 처리
- **채택 방향**: `WrongAnswersFeature`에 `favoritesItems: [FavoriteItem]` 의존성을 주입하지 않고, `FavoritesView` 레이어에서 `wrongAnswersFeature`와 `feature.items`를 함께 보유하므로 **View 레이어에서 조인**: `WrongAnswer.articleUrl`로 `feature.items`에서 tagId를 lookup → `availableTags` 계산도 View에서 수행

## 구현 순서

1. **ST-1** — `FavoritesFeature`에 `selectedTagId: UUID?` 상태, `selectTag(_ id: UUID?)` 액션, `filteredItems: [FavoriteItem]` computed property 추가. `allTags: [Tag]` 프로퍼티도 외부 주입용으로 추가 (loadFavorites 내에서 items의 tagId 기반으로 availableTags 갱신)
2. **ST-3** — `WrongAnswersFeature`에 `selectedTagId: UUID?` 상태, `selectTag(_ id: UUID?)` 액션 추가. filteredItems는 View에서 FavoriteItem 조인으로 계산하므로 Feature에는 raw `items`만 유지
3. **ST-2** — `FavoritesView` 기사 탭(`articlesContent`) 상단에 `TagChipBarView` 삽입. `itemList`를 `feature.filteredItems` 기반으로 변경. `FavoritesView` init에 `allTags: [Tag]` 파라미터 추가
4. **ST-4** — `FavoritesView` 오답 탭(`wrongAnswersContent`) 상단에 `TagChipBarView` 삽입. `WrongAnswer.articleUrl`로 `feature.items`에서 tagId 조인하는 View 헬퍼 추가. `wrongAnswersList`를 필터된 결과로 변경
5. **ST-5** — `AppDependencies.swift`에서 `FavoritesView` 생성 호출부에 `allTags` 파라미터 연결. `TagPort.fetchAllTags()` 결과를 FeedFeature 초기화 시 함께 로드해 전달하거나, `FavoritesView`가 직접 TagPort를 받도록 조정
6. **ST-6** — FavoritesFeature: `selectedTagId 초기값 nil`, `selectTag 후 filteredItems 변경`, `전체 선택 시 전체 반환`, `tagId=nil 기사는 태그 선택 시 제외` 케이스 테스트
7. **ST-7** — WrongAnswersFeature: `selectTag 후 selectedTagId 변경`, `clearTag 후 nil 복원` 케이스 테스트

## 구현 세부 사항

### ST-1: FavoritesFeature 변경 내용

```swift
// 추가할 상태
private(set) var selectedTagId: UUID? = nil
var allTags: [Tag] = []  // FavoritesView에서 주입

// computed: items에서 tagId 집합 추출 → allTags 필터
var availableTags: [Tag] {
    let usedIds = Set(items.compactMap(\.tagId))
    return allTags.filter { usedIds.contains($0.id) }
}

// computed: selectedTagId로 필터
var filteredItems: [FavoriteItem] {
    guard let id = selectedTagId else { return items }
    return items.filter { $0.tagId == id }
}

// 액션
func selectTag(_ id: UUID?) {
    selectedTagId = id
}
```

### ST-3: WrongAnswersFeature 변경 내용

// 추가할 상태
private(set) var selectedTagId: UUID? = nil

// 액션
func selectTag(_ id: UUID?) {
    selectedTagId = id

### ST-4: FavoritesView 오답 탭 View 헬퍼

```swift
// FavoritesView 내 헬퍼
private func tagId(for wrongAnswer: WrongAnswer) -> UUID? {
    feature.items.first { $0.url == wrongAnswer.articleUrl }?.tagId
}

private var wrongAnswerAvailableTags: [Tag] {
    let usedIds = Set(wrongAnswersFeature.items.compactMap { tagId(for: $0) })
    return feature.allTags.filter { usedIds.contains($0.id) }
}

private var filteredWrongAnswers: [WrongAnswer] {
    guard let id = wrongAnswersFeature.selectedTagId else { return wrongAnswersFeature.items }
    return wrongAnswersFeature.items.filter { tagId(for: $0) == id }
}
```

### 태그 없는 기사 처리

- `tagId == nil`인 `FavoriteItem` → 특정 태그 선택 시 목록에서 제외, "전체" 선택 시 포함
- `WrongAnswer.articleUrl`이 `feature.items`에 없는 경우 → tagId nil로 처리 (특정 태그 선택 시 제외)
