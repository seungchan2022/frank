## 리뷰 유형

### 1. 문서 리뷰 (step-5)
- 현재 문서만으로 코드 구현 가능한가?
- 스코프가 명확한가?
- 누락된 정보가 없는가?

### 2. 코드 리뷰 (step-7)

#### 기능 검증
- [ ] 문서와 코드 일치 (오버스펙/누락 없음)
- [ ] 엣지 케이스 처리
- [ ] 에러 처리 적절

#### 아키텍처 준수
- [ ] 레이어 구조 준수
- [ ] 단방향 의존 (상향 참조 금지)

#### 코드 품질
- [ ] 타입 적용
- [ ] 중복 코드 없음
- [ ] 보안 위험 없음

### 3. 룰즈 검증 (step-2)
- `rules/0_CODEX_RULES.md` 강제 규칙 기준
- `rules/sub/` 서브 룰북 기준으로 검증
