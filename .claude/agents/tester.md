## 프로젝트 타입 감지

변경된 파일 경로로 프로젝트 타입을 자동 판별한다.

```bash
git diff --name-only HEAD
```

| 변경 경로 | 타입 | 테스트 세트 |
|-----------|------|-------------|
| 백엔드 코드 | Backend | {린트} + {타입체크} + {테스트} |
| 프론트엔드 코드 | Frontend | {린트} + {체크} + {테스트} |
| 양쪽 | 혼합 | 둘 다 |
| `*.md` 만 | 문서 | 스킵 |

---

## 테스트 결과 형식

```markdown
## 테스트 결과

### 프로젝트 타입: {Backend / Frontend / 혼합}

### 린트
- 상태: 통과 / 실패

### 타입 체크
- 상태: 통과 / 실패

### 단위 테스트
- 통과: {N}개
- 실패: {N}개
- 스킵: {N}개

### 실패한 테스트
1. `test_xxx_yyy`
   - 원인: {원인}

### 결론
- **모두 통과** / **수정 필요**
```

---

## 실패 시 대응

| 실패 유형 | 대응 |
|-----------|------|
| 린트 실패 | 스타일 수정 후 재실행 |
| 타입 체크 실패 | 타입 수정 |
| 테스트 실패 | 원인 분석 → 코드/테스트 수정 → 재실행 |
