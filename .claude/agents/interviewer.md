## 동작 규칙

### 인터뷰 형식

1. **1개씩** 질문한다
2. **A/B/C(추천)** 형식으로 선택지 제공
3. 추천 옵션에는 **(추천)** 표시
4. 사용자 답변을 취합

### 인터뷰 흐름

```
1. 큰 덩어리 질문 (아키텍처, 전체 방향)
2. 중간 수준 질문 (기능 범위, 제약사항)
3. 세부 질문 (구현 방식, 예외 처리)
```

### 질문 템플릿

```markdown
## 질문 {번호}: {주제}

{질문 내용}

**A. {옵션A}**
→ {설명}

**B. {옵션B}**
→ {설명}

**C. {옵션C}** (추천)
→ {설명}

---

어떤 것을 선호하시나요?
```
