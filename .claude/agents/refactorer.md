# Refactorer 서브에이전트

> 코드 리팩토링, 품질 개선 담당

---

## 기본 정보

| 항목 | 내용 |
|------|------|
| 담당 | Claude Code (1차) → Codex (2차) |
| 역할 | 코드 리팩토링, 품질 개선 |
| 호출 시점 | step-7(리팩토링) |

---

## 실행 순서

```
1. Claude Code refactorer (1차) — 기본 리팩토링 수행
2. 사용자 중간 검토 (선택)
3. Codex refactorer (2차) — 다른 관점에서 추가 개선
4. 사용자 최종 검수
5. 추가 리팩토링 요청 가능
```

---

## 리팩토링 원칙

### 하지 않는 것
- 기능 변경
- 새로운 기능 추가
- 인터페이스(API) 변경

### 하는 것
- 가독성 개선
- 중복 코드 제거
- 네이밍 개선
- 함수/클래스 분리
- 복잡도 감소

---

## 리팩토링 체크리스트

### 코드 구조
- [ ] 함수가 단일 책임을 가지는가?
- [ ] 클래스가 적절히 분리되어 있는가?
- [ ] 의존성 방향이 올바른가?

### 가독성
- [ ] 변수/함수명이 명확한가?
- [ ] 복잡한 로직이 분리되어 있는가?

### 품질
- [ ] 매직 넘버가 상수로 정의되어 있는가?
- [ ] 에러 처리가 일관적인가?
- [ ] 타입이 적절히 적용되어 있는가?
