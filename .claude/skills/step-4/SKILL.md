# Step 4: 서브태스크 인터뷰

## 수행 작업

```
[1] 의존성이 해결된 서브태스크 선택
       ↓
[2] A/B/C(추천) 형식으로 1개씩 상세 인터뷰
       ↓
[3] 큰 덩어리 → 작은 세부사항 순으로 정의
       ↓
[4] 우선순위 및 구현 순서 결정
       ↓
[5] 서브태스크 문서 생성/업데이트
```

## 인터뷰 규칙
- **1개씩** 질문한다
- **A/B/C(추천)** 형식으로 선택지 제공
- 추천 옵션에는 **(추천)** 표시

## 기본 질문

- 이 서브태스크의 구체적인 목표는?
- 어떤 파일/모듈을 수정해야 하나요?
- 테스트 방법은 무엇인가요?
- 예외 케이스가 있나요?
