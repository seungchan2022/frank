# 메모 정리 + 흐름도 갱신 (/notes)

결과물은 항상 `history/개념정리.md` 하나에 통합한다.

---

## 모드 A — `/notes [메모]` (평소 작업 중)

메모를 읽어서 개념정리.md에 녹여넣는다.

### 입력 형식 (자유롭게 섞여도 됨)

```
[?] — 모르는 것 / 나중에 공부할 개념
[팁] — 멘토 또는 Claude 꿀팁
[버그] — 발견된 버그 / 기능 이상
[나중에] — 지금 말고 이후에 할 것
기타 — 분류 모르겠는 것
```

태그 없이 그냥 텍스트여도 내용 보고 분류한다.

### 처리 방식

1. `history/개념정리.md` 읽기
2. 메모 분류
   - `[?]` → 개념 설명 추가 (iOS 비유 우선, 이 프로젝트에서의 쓰임 포함)
   - `[팁]` → 꿀팁 섹션에 추가
   - `[버그]` → 버그 목록에 체크박스로 추가
   - `[나중에]` → 할 일 목록에 체크박스로 추가
   - 기타 → 내용 보고 적절한 섹션에 배치
3. 새로운 개념이면 해당 섹션에 추가, 기존 개념이면 보완
4. 개념정리.md 업데이트

### 출력

```
## 업데이트 완료

추가된 개념: N개
보완된 개념: N개
버그 추가: N개
할 일 추가: N개
```

---

## 모드 B — `/notes flow [mvp명]` (MVP 완료 시)

예: `/notes flow mvp4`

MVP 이름을 생략하면 `history/` 폴더에서 가장 최근 MVP 폴더를 자동 감지한다.
감지가 애매하면 사용자에게 확인 후 진행한다.

이번 MVP에서 달라진 흐름을 분석해서 두 곳에 저장한다.
- `history/{mvp폴더}/{날짜}_흐름도.md` — 이번 MVP 스냅샷 (마일스톤별 상세)
- `history/개념정리.md` — 최신 전체 흐름도로 갱신

### 처리 방식

1. **현재 상태 파악** (병렬)
   ```bash
   git log --oneline -30
   ```
   - `history/{현재MVP}/` 폴더 문서 전체 읽기 (마일스톤별 내용 파악)
   - `history/개념정리.md` 기존 흐름도 읽기
   - 이전 MVP 흐름도 스냅샷 읽기 (비교용)

2. **마일스톤별 변경 분석**
   - 각 마일스톤(M1, M2, M3...)에서 추가/변경된 것 식별
   - 어떤 서비스/연결/기능이 어느 마일스톤에서 생겼는지 매핑

3. **MVP 스냅샷 파일 생성** — `history/{mvp폴더}/{날짜}_흐름도.md`

   아래 구조로 작성:

   ```markdown
   # {MVP명} 흐름도 스냅샷

   ## 마일스톤별 변경 내역

   | 마일스톤 | 추가된 것 | 변경된 것 |
   |---------|---------|---------|
   | M1 | ... | ... |
   | M2 | ... | ... |

   ## 전체 흐름도 (MVP 완료 시점)

   [Mermaid 전체 흐름 다이어그램]
   - 각 노드에 어느 마일스톤에서 추가됐는지 주석 표시
     %% M1 추가
     %% M3 추가 — Apple 로그인

   ## 마일스톤별 흐름 변화

   ### M1 이후
   [Mermaid 다이어그램 — M1까지의 흐름]

   ### M2 이후
   [Mermaid 다이어그램 — M2까지 누적 흐름]

   ...각 마일스톤마다 반복
   ```

4. **개념정리.md 갱신**
   - 전체 흐름도를 최신 상태로 업데이트
   - MVP 변경 이력 표에 이번 MVP 행 추가
   - 마일스톤 상세는 스냅샷 파일 링크로 연결

   ```markdown
   ## MVP별 흐름 변경 이력
   | MVP | 마일스톤 수 | 주요 변경 | 스냅샷 |
   |-----|-----------|---------|--------|
   | MVP3 | M1~M4 | Apple 로그인, API 통합 | [링크](mvp3/흐름도.md) |
   ```

5. **completion 파일에 링크 추가**
   - `history/{mvp폴더}/` 에서 `*completion_retro.md` 파일 찾기
   - 파일 끝에 아래 섹션 추가 (이미 있으면 업데이트)

   ```markdown
   ## 흐름 변경 요약
   - {이번 MVP에서 달라진 핵심 항목 2~3개}

   → [흐름도 전체 보기]({날짜}_흐름도.md)
   ```

### 출력

```
## 흐름도 갱신 완료

스냅샷: history/{mvp폴더}/{날짜}_흐름도.md
  마일스톤: M1~MN
  추가된 것: ...
  변경된 것: ...

개념정리.md 최신 흐름도 갱신 완료
```

---

## 공통 규칙

- 개념 설명은 핵심만 간결하게 (길게 쓰지 않기)
- iOS 비유가 억지스러우면 생략
- 중복 내용은 합치기
- 개념정리.md가 없으면 새로 생성
