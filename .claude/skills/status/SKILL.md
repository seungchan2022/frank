# 워크플로우 상태 (/status)

## 수행 작업

1. 현재 진행 중인 메인태스크 확인
2. 현재 단계 및 진행률 표시
3. 서브태스크 현황 표시
4. 다음 할 일 안내

## 출력 형식

```markdown
# 워크플로우 상태

## 메인태스크
- 이름: {메인태스크명}
- 문서: progress/{YYMMDD}_{태스크명}.md

## 진행 상황
- 현재 단계: step-{N} ({단계명})
- 진행률: {N}/9 단계

## 서브태스크 현황
| # | 서브태스크 | 상태 | 의존성 |
|---|-----------|------|--------|

## 다음 할 일
→ /step-{N+1} 또는 /next
```
