# 워크플로우 상태 (/status)

## 수행 작업

1. 마일스톤 로드맵 상태 확인 (있으면) — 아이템 유형별 진행 현황 포함
2. 현재 진행 중인 메인태스크 확인
3. 현재 단계 및 진행률 표시
4. 서브태스크 현황 표시
5. 다음 할 일 안내

## 출력 형식

```markdown
# 워크플로우 상태

## 마일스톤 로드맵 (있는 경우)
- 프로젝트: {프로젝트명}
- 전체 진행률: {완료}/{전체} 마일스톤
- 현재 마일스톤: M{N} {마일스톤명}
- 로드맵: progress/milestones/{프로젝트명}/_roadmap.md

### 현재 마일스톤 아이템 현황
| # | 아이템 | 유형 | 실행 스킬 | 상태 |
|---|--------|------|----------|------|
| 1 | {아이템} | feature | /workflow | 진행중 |
| 2 | {아이템} | decision | /debate | 완료 |
| 3 | {아이템} | chore | 직접 실행 | 대기 |

## 메인태스크 (feature 아이템 실행 중인 경우)
- 이름: {메인태스크명}
- 문서: progress/{YYMMDD}_{태스크명}.md

## 진행 상황
- 현재 단계: step-{N} ({단계명})
- 진행률: {N}/9 단계

## 서브태스크 현황
| # | 서브태스크 | 상태 | 의존성 |
|---|-----------|------|--------|

## 다음 할 일
→ {유형에 따른 다음 액션 안내}
```
