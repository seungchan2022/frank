# 서브에이전트 사용 규칙 (v2.0)

## 1) 목적
`Agent` 툴(Task 툴)로 서브에이전트를 호출할 때 프로젝트 규칙 준수를 강제하고 일관된 품질을 유지한다.

## 2) 사용이 권장되는 시나리오
- **병렬 탐색/분석**: 독립 레이어(서버/웹/iOS) 동시 조사 (`feedback_parallel_agents.md`)
- **병렬 테스트 실행**: 모듈별 독립 테스트
- **대규모 리팩토링**: 10개 이상 파일에 걸친 패턴 추출/구조 변경
- **문서 자동 생성**: Progress 문서, 테스트 리포트, API 문서
- **품질 검증 자동화**: 린트 + 타입체크 + 아키텍처 규칙 병렬 검증

## 3) 사용이 금지되는 시나리오
- **룰북 수정**: `rules/0_CODEX_RULES.md`, `rules/sub/*.md` 수정은 수동 검토 필수
- **프로덕션 배포**: 수동 검증 필수
- **시크릿 관리**: env, credentials, API keys 자동 처리 금지
- **데이터베이스 DDL 변경**: 스키마 변경은 수동 검토 필수

## 4) 실제 등록된 에이전트 (`.claude/agents/`)

| 에이전트 | 용도 | 모드 |
|---|---|---|
| `reviewer` | 문서/코드 리뷰, rules/ 준수 검증 | delegate |
| `refactorer` | 코드 리팩토링 (Codex 2차와 연계) | delegate |
| `tester` | 테스트 실행 및 결과 수집 + KPI 수치 수집 | delegate |
| `interviewer` | 요구사항 A/B/C 인터뷰 | delegate |
| `git` | 커밋 생성 (푸시 금지, KPI 게이트 통과 후) | delegate |
| `deep-analysis` | 심층 코드/아키텍처 분석 | plan |
| `debate` | 3자 토론 (Claude + Codex + Serena) | plan |

내장 에이전트 유형:
- `Explore`: 코드베이스 탐색 전용 (코드 수정 금지)
- `Plan`: 설계 플랜 수립 (승인 전 수정 금지)
- `general-purpose`: 범용 다단계 태스크

## 5) 모드 선택 기준

**plan 모드 필수**:
- 10개 이상 파일 수정
- 공통/shared 모듈 변경
- 새로운 패턴/추상화 도입
- 계층 구조 변경
- 보안 관련 변경

**delegate 모드 허용**:
- 테스트 실행 및 결과 수집
- 린트/타입체크 실행
- 문서 생성
- 버그 분석 (수정 제외)
- 코드베이스 탐색

## 6) 금지된 에이전트-작업 조합

| 에이전트 | 금지 작업 |
|---|---|
| 모든 에이전트 | 룰북 수정 (`rules/*`) |
| 모든 에이전트 | 프로덕션 배포 |
| 모든 에이전트 | 시크릿/credentials 처리 |
| delegate 모드 | 아키텍처 변경 (plan 필수) |
| Explore | 코드 수정 |

## 7) 결과 검증 체크리스트

서브에이전트 작업 완료 후 반드시 확인:
- [ ] 테스트 통과
- [ ] 린트/타입체크 통과
- [ ] 아키텍처 패턴 준수 (에코 서버 + 포트/어댑터)
- [ ] 계층 의존성 위반 없음 (`api → services → domain ← infra`)
- [ ] **active MVP의 Hard KPI 지표 미달 없음** (`scripts/kpi-check.sh`)

## 8) 절대 금지

- 서브에이전트에게 룰북 수정 권한 부여
- 수동 검증 없이 프로덕션 배포
- 시크릿 자동 생성/수정
- 검증 체크리스트 생략
- KPI Hard 게이트 우회 (`KPI_BYPASS=1`)를 사유 기록 없이 사용
