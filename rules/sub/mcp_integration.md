# MCP 사용 정책 (v2.0)

이 문서는 MCP 서버의 안전 제한, 비용 제한, 핵심 우선순위 체인을 정의한다.

---

## 1. 안전 제한

| MCP | 제한 | 이유 |
|-----|------|------|
| Chrome DevTools | **로컬/스테이징만** 접근 | 프로덕션 접근 금지 |
| Playwright | **로컬/스테이징만** 접근 | 프로덕션 접근 금지 |
| Supabase | DDL 변경(DROP/ALTER) **사전 승인 필요** | 데이터 안전 |
| Supabase | 프로덕션 데이터 직접 수정 금지 | 데이터 안전 |
| Firecrawl | 크롤링 대상 **사전 승인 필요** | 불필요한 외부 접근 방지 |
| Memory | 민감 정보(API 키, 비밀번호, 토큰) 기록 금지 | 보안 |

## 2. 비용 제한

| MCP | 제한 | 이유 |
|-----|------|------|
| Context7 | `query-docs` 최대 **3회/태스크** | API 호출 비용 절감 |
| Codex | **항상 2차 패스**로만 사용 (Claude 1차 분석 이후) | 검증 전용 |

## 3. 핵심 우선순위 체인 (참고용)

### 코드 분석
```
Serena (심볼 분석) → Grep/Glob (패턴 검색) → Read (직접 읽기)
```

### 외부 문서
```
Context7 (라이브러리 공식 문서) → Tavily (기술 검색) → WebFetch (단일 URL)
```

### 웹 검색
```
Exa (시맨틱) → Tavily (키워드) → WebSearch (일반)
```

## 4. 금지 사항

- MCP 서버를 **우회하여 직접 API 호출** 금지
- **프로덕션 환경**에 접근하는 MCP 호출 금지
