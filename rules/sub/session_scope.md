# Session Scope (세션 범위 제한)

> 동시 여러 Claude Code 세션에서 모듈/패키지별로 작업 영역을 격리하기 위한 규칙.

## SS1 원칙

- 각 세션은 **1~3개 모듈/패키지**에만 집중한다
- 세션 시작 시 스코프를 명시적으로 선언한다
- 스킬/에이전트는 선언된 스코프 내에서만 코드를 분석/수정한다
- 스코프 밖 코드는 **읽기만** 허용 (인터페이스 확인 용도)

## SS2 스코프 선언 방법

### 방법 1: `/init` 시 선언 (권장)

```
/init
> 이 세션의 작업 범위: auth-module, shared-utils
```

### 방법 2: 대화 중 선언

```
이 세션은 payment-service에서만 작업한다
```

### 방법 3: 첫 메시지에 모듈명 포함

```
auth-module의 JWT 검증 기능을 구현해줘
```

→ Claude는 자동으로 해당 모듈 + 공통 모듈을 스코프로 설정

## SS3 스코프 형식

```
SESSION_SCOPE: [모듈1, 모듈2, ...]
```

예시:
- `SESSION_SCOPE: [auth-module, shared-utils]`
- `SESSION_SCOPE: [frontend-app]`
- `SESSION_SCOPE: [api-gateway, user-service, shared-types]`

> 공통/유틸 모듈은 어떤 스코프에든 암묵적으로 포함 가능

## SS4 스코프 가드 (스킬에서 준수)

### 스킬 실행 전 체크

1. 현재 세션의 `SESSION_SCOPE`가 선언되었는지 확인
2. 미선언 시: "이 세션의 작업 범위를 알려주세요" 질문
3. 선언 시: 해당 모듈 디렉토리로 작업 한정

### 스코프 내 작업 (허용)

- 스코프 내 모듈 파일 읽기/쓰기/수정
- 스코프 내 모듈 테스트 읽기/쓰기/실행
- 공통 모듈 읽기 (항상 허용)
- `rules/`, `progress/` 읽기/쓰기

### 스코프 외 작업 (제한)

- 다른 모듈 파일 **읽기만** 허용 (인터페이스/타입 확인)
- 다른 모듈 파일 **수정 금지** -- 다른 세션이 담당
- 스코프 외 모듈의 테스트 실행 **금지**

### 스코프 위반 시

```
WARNING: {파일}은 현재 세션 스코프({SESSION_SCOPE}) 밖입니다.
다른 세션에서 작업 중일 수 있습니다. 계속하시겠습니까?
```

## SS5 특수 케이스

### 공통 모듈 수정

- 공통/shared 모듈은 모든 패키지가 의존하므로 **전용 세션**에서 수정 권장
- 다른 모듈 세션에서 수정 시 반드시 사용자 확인

### 스키마/타입 정의 변경

- 공유 스키마/타입 수정은 **영향받는 모든 모듈에 영향**
- 반드시 사용자 확인 후 수정

### progress/ 문서

- 동일 메인태스크 문서를 여러 세션에서 동시 수정 시 충돌 가능
- 서브태스크별로 분리된 문서 사용 권장: `progress/subtask/{YYMMDD}_{제목}/`

## SS6 스코프 영속화 (CLAUDE.local.md)

> 스킬 실행 시 컨텍스트 압축으로 SESSION_SCOPE가 소실될 수 있다.
> 이를 방지하기 위해 **CLAUDE.local.md**에 스코프를 영속화한다.

### 설정 방법

각 터미널/워크트리 루트에 `CLAUDE.local.md`를 생성:

```markdown
# Session Scope
SESSION_SCOPE: [auth-module, shared-utils]
이 세션은 위 모듈만 수정한다. 스킬 실행 후에도 이 스코프를 유지한다.
```

- `CLAUDE.local.md`는 `.gitignore`에 자동 추가됨 → 터미널별 독립 운영 가능
- **매 대화 시작 시 자동 로드** → 컨텍스트 압축에도 생존
- 스킬 완료 후 Claude가 스코프를 잊어버리면 CLAUDE.local.md에서 복원

## SS7 git 충돌 방지

- 같은 파일을 여러 세션에서 동시 수정하면 git 충돌 발생
- 세션별 커밋 전 `git status`로 다른 세션의 미커밋 변경 확인
- 충돌 발견 시 사용자에게 보고하고 대기
