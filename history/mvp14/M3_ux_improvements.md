## 작업 순서 원칙

**서버/DB 먼저 → iOS + 웹 독립 병렬** 순서로 진행.
DEBT-04~05는 iOS 전용이므로 바로 진행. DEBT-06~07은 iOS+Web 공통이므로 서버 변경 없이 iOS·웹 병렬로 진행.

## 워크플로우 진입점

```
/workflow "M3-UX 개선: DEBT-04 좋아요 터치 분리 + DEBT-05 태그 스와이프 + DEBT-06 요약 하단 분리 + DEBT-07 카드 구분"
```

**메인태스크**: 피드 좋아요 버튼 터치 분리, 태그 스와이프 탐색 + 삭제 롱프레스 전환, 요약/인사이트 하단 영역 분리, 기사 소개 vs AI 요약 카드 UI 구분 — 4건의 UX 마찰을 제거한다.

- DEBT-04(좋아요 터치) → iOS FeedView 버튼 제스처 분리 (simultaneousGesture 또는 onTapGesture 버블링 차단)
- DEBT-05(태그 스와이프) → 태그 바 SwipeGesture 추가, 스크랩/오답 삭제를 .contextMenu 또는 EditMode로 전환
- DEBT-06(요약 하단) → 요약/인사이트 콘텐츠를 ScrollView 하단 또는 Sheet로 분리, 액션 버튼은 고정 위치 유지
- DEBT-07(카드 구분) → 배경색·테두리·헤더 레이블(기사 소개 / AI 요약 / 인사이트)로 카드 컴포넌트 분리
