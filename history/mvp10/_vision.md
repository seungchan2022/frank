# 비전: Frank MVP10

> 생성일: 260414
> 진입점: MVP9 → MVP10

## 씨앗

MVP9까지 기능 구현 완료 후 실제 테스트에서 발견한 버그들과 성능 문제 해결.
신규 기능 없음 — 기존 기능을 제대로 동작하게 만드는 것이 목표.

**확인된 문제:**
- iOS 퀴즈 완료 화면에 다시 풀기 / 오답 보기 버튼 없음 (웹에는 있음)
- iOS 즐겨찾기 목록에서 퀴즈 완료 배지가 표시되지 않음 (quizCompleted false)
- 좋아요 기반 keyword_weights가 태그 변경 후에도 피드 검색 쿼리에 붙어 오염
- 요약 실패 시 사용자에게 기술적 에러 메시지 그대로 노출
- 피드 매 호출마다 외부 검색 API 재호출로 응답 느림

## 핵심 문제

"완성된 것처럼 보이지만 실사용에서 부서지는 기능들을 고쳐 실제로 완성한다."

## 타겟 사용자

기존 Frank 사용자 — 퀴즈, 피드, 즐겨찾기를 쓰는 모든 사용자.

## 성공 기준

- iOS 퀴즈 완료 화면에서 다시 풀기 / 오답 보기 버튼이 작동함
- iOS 즐겨찾기에서 퀴즈 완료 배지가 정상 표시됨
- 태그를 바꿔도 이전 태그 키워드가 새 피드에 영향을 주지 않음
- 요약 실패 시 "요약을 불러오지 못했습니다." 같은 친화적 메시지가 표시됨
- 피드 첫 로딩 응답이 TTL 캐시 도입으로 체감 개선됨
