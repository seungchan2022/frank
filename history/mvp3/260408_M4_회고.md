## 주요 결정 사항

### 1. use:enhance 바깥 별도 form
OAuth redirect는 서버에서 302 redirect를 던지는데, SvelteKit enhanced form이 이를 fetch로 가로채 top-level navigation이 되지 않는다.
→ Apple 버튼만 `use:enhance` 없는 별도 `<form method="POST">` 로 분리.

### 2. Supabase Client IDs 순서
Supabase는 Client IDs 첫 번째 값을 OAuth `client_id`로 사용한다.
Apple은 Bundle ID(`dev.frank.app`)를 Service ID로 인식하지 않음.
→ `com.frank.web,dev.frank.app` 순서로 설정 (Service ID 먼저).

### 3. Hide My Email — 경고 불필요
`@privaterelay.appleid.com` 릴레이 주소는 Apple이 정상 제공하는 개인정보 보호 기능.
실제 이메일 없이도 Supabase 계정 생성 및 로그인 가능.
→ ST-W3(경고 메시지 UI) 삭제.

### 4. 크로스 플랫폼 계정 연동
동일 Apple ID → 동일 Supabase user_id → profiles/태그 데이터 자동 공유.
웹에서 온보딩 완료 후 iOS 로그인 시 온보딩 스킵 확인.

### 5. 서버 변경 없음
Supabase Auth가 발급한 JWT는 플랫폼(웹/iOS)에 관계없이 동일 구조.
Rust 서버의 JWT 검증 로직 재사용 가능.
