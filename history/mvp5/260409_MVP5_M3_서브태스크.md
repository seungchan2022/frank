> 브랜치: `feature/260409_mvp5_m3_favorites`

---

## 메인태스크 요약

기사 상세 페이지에서 즐겨찾기 추가/해제 토글을 구현하고, 스크랩 목록 탭을 신규 추가한다.
즐겨찾기는 favorites DB에 기사 메타 전체를 저장하며, 요약 시 자동으로 summary/insight가 채워진다.

---

## 확정 사항 (step-4 인터뷰 + 코드베이스 분석 결과)

| 항목 | 결정 |
|------|------|
| 서버 POST /me/favorites | INSERT, 중복 시 409 Conflict |
| 서버 DELETE /me/favorites | query param `?url=<encoded>` |
| 서버 GET /me/favorites | ORDER BY created_at DESC |
| FavoritesPort 확장 | 기존 `update_favorite_summary` 보존 + M3 메서드 3개 추가 |
| iOS 탭 구조 | 현재 TabView 없음 → M3에서 TabView 신규 추가 (피드 탭 + 스크랩 탭) |
| iOS FavoritesPort | Swift protocol, `FavoritesPort.swift` 신규 |
| iOS 즐겨찾기 상태 관리 | `FavoritesFeature`가 `likedUrls: Set<String>` 보유 |
| iOS FavoritesFeature 공유 | `MainTabView`에서 `@State` 1개 생성 → 피드/스크랩 탭 양쪽에 파라미터로 주입 |
| 웹 즐겨찾기 타입 | `Favorite` interface → `web/src/lib/types/favorite.ts` 신규 |

| 웹 loadFavorites 호출 시점 | store 내부에 guard (`if favorites.length > 0 return`) — 호출처 조건 불필요 |
| 웹 /favorites 라우트 | 신규 `/favorites/+page.svelte` |
| 웹 /feed/article 즐겨찾기 버튼 | M2 예약 버튼 활성화, 디테일 페이지에만 배치 |

  ```rust
  fn add_favorite<'a>(
      &'a self,

      item: &'a Favorite,
  ) -> Pin<Box<dyn Future<Output = Result<Favorite, AppError>> + Send + 'a>>;

  fn delete_favorite<'a>(
  ```

- **산출물**: M3 메서드 3개가 추가된 `FavoritesPort` trait

### ST2 — 서버 FavoritesAdapter M3 확장 (Fake + Postgres)
- **목적**: `FavoritesPort` M3 메서드 구현체 2종 완성
- **파일**:
  - `server/src/infra/fake_favorites.rs` — `add_favorite`, `delete_favorite`, `list_favorites` 추가
  - `server/src/infra/postgres_favorites.rs` — 동일 3메서드 sqlx 구현
- **Fake 구현 상세**:
  - `store: HashMap<(Uuid, String), Favorite>` 로 확장 (현재 `SummaryStore` 타입 교체)
  - `add_favorite`: 키 이미 존재 시 `AppError::Conflict` (409 매핑용 신규 에러 변형 필요)
  - `delete_favorite`: 없어도 `Ok(())`
  - `list_favorites`: 저장 순서 역순으로 반환 (created_at DESC 모사)
- **Postgres 구현 상세**:
  - `add_favorite`: `INSERT INTO favorites ... RETURNING *`, UNIQUE 위반 시 `AppError::Conflict`
  - `delete_favorite`: `DELETE FROM favorites WHERE user_id=$1 AND url=$2`
  - `list_favorites`: `SELECT * FROM favorites WHERE user_id=$1 ORDER BY created_at DESC`
- **신규 에러 변형**: `server/src/domain/error.rs`에 `AppError::Conflict(String)` 추가 → HTTP 409 매핑
- **완료 기준**: `cargo clippy -- -D warnings` + `cargo test` 통과
- **예상 시간**: ~50분

### ST3 — POST/DELETE/GET /me/favorites 핸들러 (TDD)
- **목적**: HTTP 레이어에서 FavoritesPort 호출, AppState에 기존 favorites Arc 활용
- **파일**:
  - `server/src/api/favorites.rs` (신규) — 핸들러 3개 + 단위 테스트
  - `server/src/api/mod.rs` — `pub mod favorites` 등록 + 라우터에 경로 추가
- **엔드포인트**:

  ```
  POST   /me/favorites        → add_favorite → 201 + Favorite JSON
  DELETE /me/favorites?url=.. → delete_favorite → 204
  GET    /me/favorites        → list_favorites → 200 + Vec<Favorite>
  ```

- **요청 바디 (POST)**:
  ```json
  {
    "url": "https://...",
    "title": "...",
    "snippet": null,
    "source": "...",
    "published_at": null,
    "tag_id": null,
    "summary": null,
    "insight": null
  }
  ```
- **테스트 케이스**:
  - POST 정상 → 201 + Favorite 반환
  - POST 중복 → 409
  - DELETE 정상 → 204
  - DELETE 없는 URL → 204 (no-op)
  - GET 목록 → 200 + 배열
  - 인증 없음 → 401
- **완료 기준**: `cargo test` 전체 통과
- **예상 시간**: ~90분

### ST6 — iOS FavoritesFeature
- **목적**: 즐겨찾기 추가/해제/목록 로딩 로직 (ViewModel)
- **파일**:
  - `ios/Frank/Frank/Sources/Features/Favorites/FavoritesFeature.swift` (신규)
- **산출물**: `@Observable FavoritesFeature`
  - 상태: `phase: FavoritesPhase` (idle/loading/done/failed), `items: [FavoriteItem]`, `likedUrls: Set<String>`
  - `likedUrls`는 `items`에서 `$derived` (computed): `Set(items.map(\.url))`
  - 액션:
    - `loadFavorites()` — GET /me/favorites → items + likedUrls 갱신
    - `addFavorite(feedItem:summary:insight:)` — POST → items에 prepend
    - `removeFavorite(url:)` — DELETE → items에서 제거
  - 즐겨찾기 여부: `func isLiked(_ url: String) -> Bool { likedUrls.contains(url) }`
- **테스트 케이스**:
  - 목록 로딩 → items + likedUrls 갱신
  - 추가 → likedUrls에 url 포함 확인
  - 삭제 → likedUrls에서 url 제거 확인
  - 중복 추가 → 에러(409) → phase.failed
  - API 실패 → phase.failed
- **완료 기준**: Swift Testing 통과
- **예상 시간**: ~90분

- ST2: `FakeFavoritesAdapter` + `PostgresFavoritesAdapter` 동시 확장 방향 — M2에서 이미 두 어댑터가 동일 패턴으로 구현됨.
- ST3: `AppState.favorites: Arc<dyn FavoritesPort>` M2에서 이미 추가됨 — 추가 DI 없이 바로 핸들러 작성 가능.
- ST5/ST6/ST7/ST8: iOS 포트/어댑터/피처 패턴 — 기존 SummarizePort/ArticleDetailFeature 구조와 일치.
- ST10: `web/src/lib/types/favorite.ts` 신설 + `ApiClient` 확장 — 기존 `summary.ts` / `client.ts` 확장 패턴과 동일.
- ST11/ST12/ST13: Svelte 5 store + 라우트 추가 — `summaryCache.svelte.ts` 패턴 재사용 타당.

**조건부 승인 항목**
- ST1 완료 기준 문제: trait에 메서드 추가 즉시 기존 impl 2개(`fake_favorites.rs`, `postgres_favorites.rs`)가 컴파일 에러를 냄. ST1과 ST2는 **원자적으로 묶어서 진행해야 함** — 각각 독립 완료 불가.
- ST3 라우터 등록 위치 오류: 서브태스크 문서에 `api/mod.rs`에 경로 추가라고 기재됨. 실제로는 `server/src/lib.rs`의 `create_router()`에 라우트를 등록해야 함. `api/mod.rs`는 모듈 export + AppState 정의만 담당.
- ST13 인증 체크 중복: `+page.ts`에 인증 체크 추가 불필요. `routes/+layout.server.ts`의 전역 가드(PUBLIC_PATHS 기반 redirect)가 이미 `/favorites`를 보호함.

**누락 항목**
- Favorites → Detail 요약 재주입 전략 미정의: `/favorites` 목록에서 상세 진입 시 `FavoriteItem.summary/insight`를 `summaryCache` 또는 `ArticleDetailFeature` 초기 상태로 주입해야 함. 현재 상세 페이지(`+page.ts`)는 `FeedItem`만 받으므로 `Favorite → FeedItem` 매핑 + 저장된 요약 재주입 전략을 ST13 또는 별도 ST에 명시해야 함.
- iOS에도 동일: `FavoritesView`에서 `ArticleDetailView` 진입 시 `SummarySessionCache.hydrate(url:result:)` 또는 `ArticleDetailFeature` 초기 `done` 상태 주입 경로가 설계에 없음.

### Codex 리뷰 (기술적 타당성)

**수정 필요 (필수)**
- `[1] ST2 Postgres Conflict 처리`: sqlx PG unique violation은 error code `23505`로 분기 필요. 에러 메시지 문자열 검사는 취약함.
- `[2] ST2 Favorite sqlx::FromRow 누락`: `INSERT ... RETURNING *`로 `Favorite` 구조체 반환 시 `#[derive(sqlx::FromRow)]` 또는 수동 매핑 필요. 현재 `models.rs`에 없음.
- `[3] ST2 Fake created_at DESC 정렬`: fake의 `list_favorites()`에서 `created_at DESC` 모사 시 `Favorite.created_at`으로 sort 또는 insertion order 별도 저장 필요.
- `[4] ST2 update_favorite_summary 의미 유지`: 기존 Fake는 "없는 URL도 insert"하지 않고 no-op. 새 HashMap 타입으로 교체 후에도 이 의미 보존 필요 (M2 요약 경로 회귀 방지).
- `[5] ST11 웹 store guard 불충분`: 설계의 `if favorites.length > 0 return` guard는 빈 즐겨찾기 사용자에서 동작 불가, 사용자 전환 시 이전 데이터 오염. `loaded: boolean` 또는 `loadedForUserId: string | null` 상태 필요.
- `[6] ST3 POST body DTO 분리`: `id`, `user_id`, `created_at`은 서버가 채워야 하므로 요청 바디에 포함 금지. 별도 `AddFavoriteRequest` struct 정의 권장.
- `[7] ST5 AppDependencies live()/mock() 동시 보강`: `favorites: any FavoritesPort` 추가 시 `live()` + `mock()` 생성자 모두 수정 필요. 현재 4개 포트만 존재.
- `[8] ST7 iOS 요약 결과 전달`: 버튼 토글 시 `phase.done`에서 summary/insight를 꺼내 `addFavorite`에 전달하지 않으면 서버에 null로 저장됨. 명시적 전달 로직 필요.

**주의 (권장 수정)**
- `[A] ST5 FavoriteItem.url 타입`: `String` vs `URL` — `FeedItem`의 url 타입 일관성 유지 필요.
- `[B] ST6 iOS loadFavorites guard`: 빈 즐겨찾기 사용자 재호출 방지를 위한 `hasLoaded: Bool` 플래그 또는 타임스탬프 비교 필요.
- `[C] ST8 설정 sheet 이동`: `FeedContainerView → MainTabView` 전환 시 기존 설정 sheet 진입점 위치 재조정 필요.
- `[D] ST3 DELETE 빈 url 처리`: 빈 문자열 query param은 204 no-op 대신 400 반환이 일관적.
- `[E] ST3 401 테스트 위치`: handler 단위 테스트만으로 401을 검증할 수 없음. `create_router` 기준 통합 테스트로 작성.
