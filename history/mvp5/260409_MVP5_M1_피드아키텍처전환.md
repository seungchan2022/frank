## DB 스키마 (확정)

### articles 테이블: 제거

기존 articles 테이블은 불필요. 관련 마이그레이션을 롤백하거나 새 마이그레이션으로 교체.

### favorites 테이블 (신규)

```sql
favorites (
  id           uuid PK default gen_random_uuid(),
  user_id      uuid NOT NULL REFERENCES profiles(id) ON DELETE CASCADE,
  title        text NOT NULL,
  url          text NOT NULL,
  snippet      text,
  source       text NOT NULL,
  published_at timestamptz,
  tag_id       uuid,
  summary      text,
  insight      text,
  liked_at     timestamptz,
  created_at   timestamptz NOT NULL DEFAULT now(),
  UNIQUE (user_id, url)
);
```

## GET /me/feed 엔드포인트 스펙

**요청**: `GET /api/me/feed` (인증 필수)

**동작**:
1. 사용자 태그 조회 (DB: get_user_tags)
2. 각 태그명으로 검색 API 호출 (SearchChainPort)
3. 결과 취합 → FeedItem 목록 반환 (DB 저장 없음)

**응답**:
```json
[
  {
    "title": "...",
    "url": "...",
    "snippet": "...",
    "source": "...",
    "published_at": "2026-04-09T00:00:00Z",
    "tag_id": "uuid"
  }
]
```

> `id` 필드 없음 — ephemeral 결과. 클라이언트는 url을 기사 식별자로 사용.
