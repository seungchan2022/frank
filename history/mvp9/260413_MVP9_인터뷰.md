> 작성일: 2026-04-13

## 수집된 이슈 전체

### 1. 퀴즈 UX / 기능

| # | 내용 | 상세 |
|---|------|------|
| Q1 | 퀴즈 완료 직후 오답 인라인 표시 | 퀴즈가 끝나면 완료 화면에서 이번 세션에 틀린 문제를 바로 보여줌. 오답 노트 탭으로 이동하지 않아도 됨. |
| Q2 | 기사 상세 퀴즈 버튼 재설계 | 퀴즈 완료 후 "퀴즈 풀기" 버튼이 `[다시 풀기]` `[오답 보기]` 두 버튼으로 변경. 오답 보기는 시트로 표시. |

**구현 난이도**
- Q1: 클라이언트 상태만으로 가능. 퀴즈 세션 중 오답 데이터를 이미 로컬에 갖고 있으므로 서버 변경 없음.
- Q2: `quiz_completed` 플래그(MVP8에서 추가)를 활용. 클라이언트 UI 변경만 필요.

---

### 2. 속도

| # | 내용 | 상세 |
|---|------|------|
| S1 | 요약 속도 개선 | 현재 30초 이상 소요. 실사용 불가 수준. |
| S2 | 퀴즈 생성 속도 개선 | 요약과 동일한 LLM 경로. 동일하게 느림. |

**원인 분석**
- 현재 모델: `qwen/qwen3.5-plus-02-11` (OpenRouter 경유)
- OpenRouter free tier는 대기열이 길어서 레이턴시가 30초+ 발생
- `server/src/infra/openrouter.rs` timeout: 60초

**해결 방향**
- Groq로 교체 (`llama-3.3-70b` 기준 3~5초 예상)
- 이전 `project_api_cost_policy.md`에서 Groq 교체 확정된 방향

### 3. 기사 / 피드

| # | 내용 | 상세 |
|---|------|------|
| A1 | 피드 초기 로딩 속도 | 첫 진입 시 기사가 뜨기까지 느림 |
| A2 | snippet 품질 심각 | 기사 소개 텍스트에 사이트 네비게이션·태그·시간표 등 노이즈 혼입 |
| A3 | 피드 다양성 부족 | 새로 고침 / 새 뉴스 가져오기를 눌러도 같은 기사가 반복해서 나옴 |

**A2 실제 사례 (스크린샷 확인)**
- `TechCrunch –:–:–:– The first StrictlyVC of 2026 hits San Francisco. Tickets are going...` — 사이트 제목과 광고 문구 혼입
- `arrow_forward - #Android Studio - #Compose - #Jetpack Navigation...` — 아이콘 이름 + 태그 목록
- `모바일 기기/앱 | PR Newswire 웹 접근성 Skip Navigation 이동 보도자료: 00:00 01:00 02:00...` — 네비게이션 + 시간표

**A2 원인**
- Tavily `content` 필드 자체가 페이지를 긁은 날것 텍스트
- MVP8 `clean_snippet`은 HTML 태그·URL·이메일만 제거 → 위 패턴은 필터링 불가
- URL params로 전달된 snippet(피드→기사상세)에는 `clean_snippet` 자체 미적용

**A3 원인 (코드 확인)**
- `server/src/infra/tavily.rs:66` — 검색 body에 `days` 파라미터 없음
- 날짜 필터 없이 같은 키워드로 검색하면 Tavily가 항상 동일한 상위 기사 반환
- `"days": 7` 한 줄 추가로 최근 7일 기사만 가져오도록 개선 가능
