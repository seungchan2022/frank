## 메인태스크 요약

M2에서 누적된 `user_keyword_weights`를 활용해 피드 검색 쿼리를 보강하고,
기사 상세 하단에 연관 기사 섹션을 제공한다.
`GET /me/feed` API 시그니처는 불변 — 서버 내부 로직만 수정.

## 주요 설계 결정

| 항목 | 결정 | 이유 |
|------|------|------|
| 피드 개인화 threshold | like_count >= 3 | M2 기획 확정값 유지 |
| 개인화 키워드 수 | top 3 (피드), top 5 (연관 기사) | 연관 기사는 태그 쿼리 없어 더 많은 키워드로 정확도 향상 |
| 연관 기사 입력 | title + snippet + top keywords | 추가 스크래핑 없음 |
| 연관 기사 실패 처리 | 빈 배열 반환 (500 미전파) | 보조 기능 — 메인 피드에 영향 없어야 함 |
| LLM 실패 처리 | soft fail (빈 배열 폴백) | rate limit 시 좋아요 카운트는 유지 |
| LikesFeature 업데이트 전략 | 비관적 (API 완료 후 상태 변경) | 테스트 명세(TDD) 준수 — 실패 시 likedUrls 변경 없음 |
