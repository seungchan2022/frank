## 서브태스크 상세

### ST-1: LlmPort + DbPort 확장

**파일**: `server/src/domain/ports.rs`

변경 내용:
- `LlmPort`에 `extract_keywords` 추가 — **`Arc<dyn LlmPort>`로 사용되므로 boxed future 필수**:
  ```rust
  fn extract_keywords<'a>(
      &'a self,
      title: &'a str,
      snippet: Option<&'a str>,  // FeedItem.snippet이 Option이므로 Option<&str>
  ) -> Pin<Box<dyn Future<Output = Result<Vec<String>, AppError>> + Send + 'a>>;
  ```
- `DbPort`에 다음 3개 메서드 추가 (DbPort는 제네릭이므로 async fn 스타일 유지 가능):
  - `increment_keyword_weights(user_id: Uuid, keywords: Vec<String>) -> Result<()>`
  - `get_top_keywords(user_id: Uuid, limit: u32) -> Result<Vec<String>>`  ← `usize` 대신 `u32`
  - `increment_like_count(user_id: Uuid) -> Result<i32>`

완료 기준: `cargo build` 통과 (기존 구현체 mock 포함 컴파일 성공)

### ST-3: PostgresDB keyword weights + like_count 구현

**파일**:
- `server/src/infra/fake_db.rs` — mock 3개 메서드 추가
- `server/src/infra/postgres_db.rs` — 실 구현 3개 메서드 추가

구현 사항:
- `increment_keyword_weights`:
  ```sql
  INSERT INTO user_keyword_weights (user_id, keyword, weight, updated_at)
  VALUES ($1, $2, 1, NOW())
  ON CONFLICT (user_id, keyword) DO UPDATE
    SET weight = user_keyword_weights.weight + 1, updated_at = NOW()
  ```
- `get_top_keywords`:
  ```sql
  SELECT keyword FROM user_keyword_weights
  WHERE user_id = $1
  ORDER BY weight DESC, updated_at DESC, keyword ASC
  LIMIT $2
  ```
  ← deterministic 정렬 (테스트 안정성 + M3 일관성)
- `increment_like_count`: `UPDATE profiles SET like_count = like_count + 1 WHERE id = $1 RETURNING like_count`
  - profile row 부재 시 AppError::NotFound 반환 (update_profile_onboarding과 일관성 유지)

완료 기준: `cargo test` 통과 (3개 메서드 단위 테스트 포함)

### ST-4: POST /me/articles/like 엔드포인트 + 서비스

**파일**:
- `server/src/services/likes_service.rs` (신규) — 오케스트레이션
- `server/src/api/likes.rs` (신규) — 핸들러 (얇게: 파싱 + 서비스 호출 + 응답 변환)
- `server/src/api/mod.rs` — 라우터 등록

엔드포인트:
```
POST /me/articles/like
요청: { "title": "...", "snippet": "..." }   ← url 제거: 서버에서 사용처 없음, 중복방지는 클라이언트 책임
처리: extract_keywords(title, snippet) → increment_keyword_weights → increment_like_count
응답: { "keywords": [...], "total_likes": N }
```

idempotency 정책: 클라이언트가 이미 liked URL 재클릭 시 요청 안 보냄 (서버는 중복 체크 없음 — 이벤트 누적 모델).

완료 기준: `cargo test` 통과 (통합 테스트 포함, 서버 196+ → 목표 202+)
