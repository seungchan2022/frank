## 잘한 것 (Keep)

### 1. 기획에 하루를 온전히 투자한 것

1일차를 코드 없이 기획에만 썼다. 경쟁 서비스 조사, 아키텍처 논쟁, 기능 범위 확정을 모두 마치고 2일차에 들어갔다. 결과적으로 구현 중 방향 전환이 한 번도 없었다. "기획이 탄탄하면 구현은 직선이다"라는 걸 체감했다.

### 2. 에코 서버 + 포트/어댑터 패턴

모든 외부 호출을 포트(trait)로 추상화하고, Fake 어댑터로 테스트한 전략이 주효했다. 특히:
- 외부 API 없이도 전체 비즈니스 로직 테스트 가능
- 새 어댑터 추가가 간단 (Firecrawl, iMessage 추가 시 trait 구현만)
- 서버 테스트 69개가 0.01초에 완료 — 피드백 루프 극단적으로 빠름

### 3. 폴백 체인 설계

Tavily→Exa→Firecrawl→arXiv 폴백 체인이 실전에서 진가를 발휘했다. Tavily 한도 소진 시 자동으로 Exa로 전환되면서 수집이 끊기지 않았다. 무료 티어 조합으로 MVP 비용 $0 달성.

### 4. 빠른 버그 대응

수집 파이프라인 가동 후 5개 실전 버그를 발견하고 당일 전부 수정했다:
- IntersectionObserver 무한 호출
- loadInitial 누락으로 빈 피드
- usedTags 누락으로 태그 필터 미표시
- 홈페이지/목록 URL 혼입
- 스니펫 미표시

### 5. 중간 의사결정 변경을 두려워하지 않은 것

기획 시 "Supabase REST API만 사용"으로 시작했지만, 실제 구현에서 REST API의 한계를 느끼고 `sqlx 직접 연결`로 전환했다. 포트/어댑터 패턴 덕분에 어댑터만 교체하면 됐고, 서비스 레이어 변경 없이 마이그레이션 완료.

## 아쉬운 것 (Problem)

### 1. 테스트 커버리지 90% 미달

CLAUDE.md에 "90% 이상" 규칙을 명시해놨지만, 실제로는 서버/웹 모두 미달 상태다.
- 서버: 서비스 레이어 테스트 부족 (collect_service, summary_service 통합 테스트 미흡)
- 웹: 컴포넌트 테스트가 Header 1개뿐. feed, onboarding, login 페이지 테스트 없음

**교훈**: TDD를 표방했지만, 속도 압박에 "일단 돌아가게 만들자"로 흘렀다. MVP2에서는 각 마일스톤 완료 기준에 커버리지 게이트를 강제해야 한다.

### 2. 코드 리뷰 이슈 23건 미해결

Step 7 리팩토링/리뷰에서 Critical 3건, Major 12건, Minor 8건이 발견됐지만, MVP 완료를 우선시하면서 대부분 미수정.

**Critical 이슈 (MVP2에서 반드시 해결)**:
- HTTP 타임아웃 미설정 → 외부 API 무응답 시 스레드 무한 대기
- auth 미들웨어 매 요청마다 Client 생성 → 성능/안정성
- 인증 가드 race condition → $effect 실행 순서 비보장

### 3. 의존 방향 위반

"api → services → domain ← infra" 원칙을 세웠지만:
- `sqlx::FromRow`가 domain/models.rs에 존재 (infra 의존이 domain에 침투)
- `SearchFallbackChain` 구체 타입이 services에서 직접 참조 (services → infra 위반)

### 4. i18n 미적용

규칙에 "i18n 키 기반 텍스트 (하드코딩 금지)"를 명시했지만, 모든 UI 텍스트가 한국어 하드코딩 상태. 나중에 영문 지원 시 전면 수정 필요.

### 5. UI 중복 코드

onboarding과 settings의 태그 선택 UI가 완전 중복. API 프록시 라우트 2개도 100% 동일. feed/+page.svelte가 283줄로 비대.

## 놀랐던 것 (Surprise)

### 1. Rust 초보가 하루 만에 프로덕션급 API 서버를 만들 수 있다

iOS 개발자가 처음 접하는 Rust로, 하루 만에 6개 외부 API를 통합한 서버를 완성했다. 포트/어댑터 패턴 + AI 페어 프로그래밍의 시너지가 예상보다 훨씬 컸다. 언어를 "마스터"하지 않아도 패턴을 알면 코드가 나온다.

### 2. 바이브 코딩에서 가장 중요한 건 "방향 감각"

코드를 직접 치는 비중은 극히 낮았지만, 매 순간 "이건 맞고 저건 아니다"라는 판단을 내려야 했다. 에코 서버 패턴을 직접 제안한 것, Supabase REST → sqlx 전환을 결정한 것, LLM 모델을 Qwen3 thinking 비활성화로 전환한 것 — 모두 내가 내린 판단이다. AI가 코드를 짜더라도, 방향은 사람이 잡아야 한다.

### 3. 실전 버그의 80%는 "빠진 것"

5개 실전 버그 중 4개가 "코드가 틀린 게 아니라 빠진 것"이었다. loadInitial 호출 누락, usedTags 누락, 홈페이지 URL 필터 누락, 스니펫 표시 누락. 코드 리뷰가 "있는 코드의 품질"을 보는 거라면, 실전 테스트는 "없는 코드를 찾는 것"이다.

### 4. 폴백 체인의 실전 가치

이론적으로 좋아 보이는 폴백 체인이 실제로 동작하는 걸 보니 설계 투자가 맞았다는 확신이 들었다. Tavily가 "홈페이지 URL"을 반환할 때 Firecrawl이 그 URL의 본문을 크롤링하는 조합이 예상 외로 강력했다.

## 아키텍처 심층 분석

### 포트/어댑터 패턴 — 검증된 것과 개선할 것

**검증된 것**:
- Fake 어댑터로 외부 API 없이 69개 서버 테스트 0.01초 완료
- 어댑터 교체 용이성: Supabase REST → sqlx 전환 시 서비스 레이어 무변경
- 새 포트 추가 자연스러움: CrawlPort, NotificationPort 추가 시 기존 코드 영향 없음

**개선할 것**:
- domain에 infra 의존성 침투 (sqlx::FromRow) → DTO 레이어 분리 필요
- SearchFallbackChain이 services에서 구체 타입 참조 → trait으로 추상화

### 데이터 흐름 분석

```
[브라우저] → [SvelteKit 프록시] → [Rust API 서버]
                                      ├→ [Supabase PostgreSQL] (sqlx)
                                      ├→ [Tavily/Exa] (검색)
                                      ├→ [Firecrawl] (크롤링)
                                      ├→ [OpenRouter → Qwen3.5] (요약)
                                      └→ [iMessage osascript] (알림)
```

**병목 지점**: 크롤링(직렬 N건)과 LLM 요약(직렬 N건)이 전체 파이프라인 지연의 90%+. MVP2에서 `tokio::join!` 또는 `futures::join_all`로 병렬화하면 체감 속도 5~10배 개선 가능.

### 프론트엔드 상태 관리

```
Supabase Auth → auth.svelte.ts ($state)
                  ↓ onAuthStateChange
              +layout.svelte
                  ↓ 리다이렉트 분기
    /login ← 미인증
    /onboarding ← 인증+온보딩 미완료  
    /feed ← 인증+온보딩 완료
```

**race condition 위험**: `$effect` 간 실행 순서가 보장되지 않아 인증 상태 전이 시 깜빡임 가능. MVP2에서 상태 머신(state machine) 패턴 도입 고려.

## MVP2 방향 제언

### 1. "소비"에서 "학습"으로

MVP1은 "좋은 콘텐츠를 자동으로 모아주는 것"을 검증했다. MVP2는 "모인 콘텐츠로 학습하는 것"이 핵심.
- 스크랩 보관함 (북마크)
- 기사 기반 퀴즈/플래시카드
- 주간 학습 리포트

### 2. 기술적 기반 강화

MVP1에서 밝혀진 기술 부채를 해결하면서:
- 병렬 크롤링/요약으로 성능 확보
- 디자인 시스템 + 다크 모드
- iOS 앱 착수 (동일 포트/어댑터 패턴)

### 3. 개인화 강화

- 읽은 기사 기반 추천 알고리즘
- 사용자 정의 검색 쿼리
- 수집 주기 자동화 (크론)

## 회고를 마치며

2일 만에 MVP1을 완주했다. iOS 개발자가 처음 접하는 Rust + Svelte + Supabase로, 6개 외부 API를 통합한 풀스택 앱을 만들었다. 완벽하지는 않다 — 테스트 커버리지는 목표 미달이고, 코드 리뷰 이슈 23건이 쌓여 있다.

하지만 "핵심 가치 검증"이라는 MVP의 본질에서는 성공이다:
- **키워드 기반 자동 수집** ✅ 동작한다
- **다중 소스 폴백** ✅ 실전에서 검증됐다
- **LLM 한국어 요약** ✅ 인사이트가 실제로 유용하다
- **매거진 피드** ✅ 읽기 편하다

MVP2는 이 기반 위에 "학습" 레이어를 올리는 것이다. 포트/어댑터 패턴이 잘 잡혀 있으니 확장에 자신감이 있다.
