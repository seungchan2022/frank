# MVP8 로드맵 — UX 개선 + 오답 아카이빙 + 퀴즈 완료 배지

> 작성일: 2026-04-12
> 브랜치 기반: `main` (MVP7 M4 완료 이후)
> 테스트 기준선: 서버 252 / 웹 183 / iOS 189+3

---

## 목표

MVP7에서 실제로 앱을 테스트하면서 발견한 UX 문제와 기능 결함을 개선한다.
"읽는 느낌"을 높이고(텍스트 정제, 로딩 표시), 퀴즈를 "학습 루프"로 완성한다(오답 아카이빙, 완료 배지).

---

## MVP8 스코프

| 순서 | 마일스톤 | 내용 | 플랫폼 |
|-----|---------|------|--------|
| 1 | M1 | DB 스키마 + 서버 기반 — 신규 테이블/컬럼 + 오답·퀴즈완료 엔드포인트 + snippet 정제 | DB + 서버 |
| 2 | M2 | UX 개선 — 연관 기사 제거, 로딩 단계 표시, 퀴즈 확인 버튼, 타이포그래피, iOS 퀴즈 버그 수정 | 웹 + iOS (병렬) |
| 3 | M3 | 오답 아카이빙 + 퀴즈 완료 배지 — 스크랩 탭 세그먼트, 오답 저장/삭제, 배지 표시 | 웹 + iOS (병렬) |

> **구현 원칙**:
> - M1 DB 스키마와 엔드포인트를 전부 확정한 뒤 M2·M3를 진행한다.
> - M2는 기존 엔드포인트만 사용하는 클라이언트 UX 개선이다 (신규 API 호출 없음).
> - M3는 M1에서 추가된 엔드포인트를 소비한다.
> - M2·M3 내부에서 웹과 iOS는 서로 독립적으로 병렬 진행한다.
> - 이전 마일스톤 코드는 수정하지 않는다. 불가피한 경우에만 허용하고 이유를 명시한다.

## 확정된 기획 사항

| 항목 | 결정 |
|------|------|
| 오답 저장 범위 | 오답만 저장 (정답은 저장 안 함) |
| 퀴즈 완료 배지 조건 | 한 번이라도 풀면 배지 표시 (점수 무관) |
| 오답 아카이빙 위치 | 스크랩 탭 내 "기사 \| 오답 노트" 세그먼트 |
| 오답 삭제 | 가능 (사용자가 직접 삭제) |
| 연관 기사 | 제거 (재설계는 다음 MVP) |
| snippet 한국어 번역 | 보류 (영어 기사 커버리지 유지) |
| snippet 정제 방식 | 코드 필터링 — HTML 태그·URL·이메일 제거 + 단어 경계 절단 |
| 로딩 표시 방식 | 타이머 기반 텍스트 자동 전환 — "요약 중… → 마무리 중… → 완료" (실제 서버 상태 반영 아님, 체감 개선용) |
| 퀴즈 답변 방식 | 선택 후 확인 버튼 클릭 시 정오 표시 (기존: 선택 즉시 표시) |

## 실행 순서 근거 (재작업 방지)

```
M1: DB 스키마 확정 (quiz_wrong_answers 테이블, favorites.quiz_completed 컬럼)
    + 서버 신규 엔드포인트 전부 (오답 CRUD, 퀴즈 완료 마킹, favorites 응답 확장)
    + snippet 정제 (서버 코드만)
     ↓ M2·M3의 모든 서버 의존성 해결
M2: 클라이언트 UX 개선 (신규 API 호출 없음 — 기존 엔드포인트만 사용)
     ↓ 퀴즈 UI 기반(확인 버튼, 버그 수정) 확립 후 M3 진행
M3: 오답 저장·조회·삭제 + 퀴즈 완료 배지 (M1 엔드포인트 소비)
```

클라이언트 원칙:
```
M2: 기존 화면 요소 제거/수정만 (연관 기사 제거, 로딩 텍스트, 확인 버튼, 타이포)
M3: 신규 화면 추가만 (오답 노트 탭, 배지 표시)
    → M3에서 M2 코드 미수정
```

## DB 스키마 변경 (M1에서 전부 확정)

### 신규 테이블: quiz_wrong_answers

```sql
CREATE TABLE quiz_wrong_answers (
    id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id       UUID NOT NULL REFERENCES profiles(id) ON DELETE CASCADE,
    article_url   TEXT NOT NULL,
    article_title TEXT NOT NULL,
    question      TEXT NOT NULL,
    options       JSONB NOT NULL,        -- ["보기A","보기B","보기C","보기D"]
    correct_index INTEGER NOT NULL,      -- 0-based
    user_index    INTEGER NOT NULL,      -- 사용자가 선택한 보기 인덱스
    explanation   TEXT,
    created_at    TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW(),
    UNIQUE(user_id, article_url, question)  -- 같은 기사+문제 중복 방지 (덮어쓰기 정책)
);

CREATE INDEX idx_qwa_user_id ON quiz_wrong_answers(user_id);
CREATE INDEX idx_qwa_user_article ON quiz_wrong_answers(user_id, article_url);
```

- RLS: 본인 데이터만 SELECT/INSERT/DELETE
- 중복 정책: 같은 기사+문제 오답은 1개만 유지 — `INSERT ... ON CONFLICT (user_id, article_url, question) DO UPDATE`
- 정답 맞혀도 자동 삭제 안 함 — 사용자가 직접 삭제

### 기존 테이블 수정: favorites

```sql
ALTER TABLE favorites ADD COLUMN IF NOT EXISTS quiz_completed BOOLEAN NOT NULL DEFAULT false;
```

- 퀴즈를 한 번이라도 완료하면 `true`로 업데이트
- 기존 즐겨찾기 행: 기본값 `false` (배지 미표시)

1. 마크다운 헤더(`#` 시작 행) 제거

4. 이메일 토큰(`@` 포함) 제거
5. 공백 정리
6. 200자 단어 경계 절단 — 단어 중간 컷 방지, 초과 시 `…` 추가

```rust
// 변경 전
cleaned.chars().take(300).collect::<String>()

// 변경 후
fn clean_snippet(s: &str) -> String { ... }  // 위 6단계 적용
```

## M3: 오답 아카이빙 + 퀴즈 완료 배지

### 목표
퀴즈 풀기를 학습 루프로 완성한다.
오답은 스크랩 탭에 아카이빙되어 복습할 수 있고,
퀴즈를 완료한 기사는 스크랩 카드에 배지가 표시된다.
웹과 iOS는 독립적으로 병렬 진행한다.

### 웹

**퀴즈 완료 배지**
- `GET /me/favorites` 응답의 `quiz_completed: bool` 필드 활용
- 스크랩 탭 기사 카드에 퀴즈 완료 배지 표시 (예: 작은 체크 아이콘 or 뱃지)
- 퀴즈 모달에서 마지막 문제 완료 시 `POST /api/favorites/quiz/done` 호출
  - 호출 성공 후 해당 카드 `quiz_completed = true` 로컬 상태 업데이트

**오답 저장**
- 퀴즈 진행 중 오답 발생 시 `POST /api/quiz/wrong-answers` 호출
  - 전달: article_url, article_title, question, options, correct_index, user_index, explanation
