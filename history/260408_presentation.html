04 · AUTHENTICATION

Apple 로그인 — iOS vs 웹

📱 iOS 흐름

1. ASAuthorizationController 요청
↓
2. Apple → idToken + nonce 반환
↓
3. Supabase SDK signInWithIdToken
↓
4. JWT → API 요청 (Bearer 헤더)

🌐 웹 흐름

1. OAuth PKCE redirect (Service ID)
↓
2. /auth/callback?code=... redirect
↓
3. code 교환 → Supabase 세션
↓
4. httpOnly 쿠키로 세션 저장

크로스 플랫폼 계정 연동이 "공짜"인 이유:
같은 Supabase 프로젝트 → 같은 Apple ID = 같은 user_id → 별도 연동 로직 불필요

04 · AUTHENTICATION

방식 비교 + 보안

항목웹iOS
방식 OAuth PKCE redirect ID Token 직접
핵심 식별자 Service ID (com.frank.web) Bundle ID (dev.frank.app)
세션 저장 httpOnly 쿠키 (JS 접근 불가) 메모리 + Keychain
XSS 방어 ✅ httpOnly → 스크립트 접근 불가 ✅ Keychain 격리
iOS 비유

웹의 httpOnly 쿠키 = Keychain에 저장하되 앱이 직접 못 읽게 막은 것
JWT 역할 동일 — "이 사람 맞아요" 증명서
