## 요구사항 (인터뷰 합의)

| 항목 | 결정 |
|------|------|
| "전체" 탭 | 포함 (tagId=nil, 첫 번째 탭). 삭제된 태그 선택 중이면 nil(전체)로 fallback |
| 수집/요약 트리거 | 단일 "새 뉴스 가져오기" 버튼 + Pull-to-refresh. collect→summarize 순차. "수집 중...→요약 중..." 단계 표시 |
| 최초 진입 (0건) | 스켈레톤 + 점진적 채움: 태그 탭+shimmer 즉시 → 수집 완료 시 제목+출처 채움(summary="요약 중...") → 요약 완료 시 summary+insight 채움 |
| 점진적 채움 플로우 | collect 호출 → 기사 fetch → summarize 호출 → 기사 재fetch (2단계 fetch) |
| 이후 진입 | 캐시된 기사 즉시 표시, 갱신은 수동 |
| 카드 정보 | title_ko(없으면 title fallback) + source + published_at + summary + insight |
| 요약 미완료 | "요약 중..." 플레이스홀더. 완료 시 summary+insight 채움 |
| title_ko=nil | 원문 title 표시 → 요약 완료 시 title_ko로 교체 |
| 페이지네이션 | 20건 단위 무한 스크롤. 20건 미만 반환 시 hasMore=false |
| 태그 탭 전환 | 탭별 캐싱. 수집/요약 완료 시 전체 캐시 무효화 → 현재 탭 재fetch |
| 수집/요약 후 갱신 | 첫 페이지 20건 재fetch |
| 중복 방지 | isCollecting 중 버튼 비활성화 + Pull-to-refresh 무시 |
| 기사 탭 동작 | Safari로 URL 오픈 (M5에서 상세 화면으로 교체 예정) |
| 설정 | 네비바 기어 아이콘 → 설정 placeholder (로그아웃 행 1개). M6 확장 대비 |
| ArticleCard | 리스트형 (구분선 분리). 제목→출처+날짜→요약→인사이트 세로 배치 |
| EmptyState | 안내 문구만 ("이 키워드의 뉴스가 아직 없습니다"). CTA 없음 |
| 패턴 | 에코 서버 (ArticlePort + CollectPort + TagPort → Mock 주입 TDD) |

## 기술 결정

| 항목 | 결정 |
|------|------|
| 기사 읽기 경로 | **Supabase 직결 (CQRS-lite)**: 읽기=Supabase PostgREST, 쓰기=Rust 서버. tags와 동일 패턴. 서버 API 수정 불필요 |
| Article 모델 확장 | titleKo, insight, snippet, summarizedAt 4개 추가. tagId를 UUID? (optional)로 변경. content 미포함 (M5에서 추가) |
| ArticlePort 시그니처 | `fetchArticles(filter: ArticleFilter)` 구조체 방식 |
| ArticleFilter | tagId: UUID?, limit: Int = 20, offset: Int = 0 |
| CollectPort 반환값 | `triggerCollect() → Int`, `triggerSummarize() → Int` (수집/요약 건수 반환) |
| AuthPort 확장 | `getAccessToken() async throws -> String` 추가 (APICollectAdapter 토큰 전달용) |
| FeedFeature 상태 구조 | 플랫 프로퍼티 + 상태 전이 헬퍼 메서드 (@Observable 프로퍼티별 tracking 최적) |
| FeedFeature 의존 | ArticlePort + CollectPort + TagPort 3개 주입 |
| TagChipBarView | M3 TagChipView 재사용 + ScrollView(.horizontal) 컨테이너. FeedFeature가 상태 관리, View는 순수 UI |
| SupabaseArticleAdapter | PostgREST로 articles 직접 쿼리 (tag_id 필터 + offset + limit + order by created_at desc). SupabaseTagAdapter와 동일 패턴 |
| 서버 URL 관리 | ServerConfig (SupabaseConfig 패턴 동일, Info.plist/Secrets.plist) |
| APICollectAdapter 인증 | AuthPort.getAccessToken()으로 Bearer 토큰 획득 |
| 정렬 기준 | created_at DESC (DB 인덱스 idx_articles_user_created 활용) |
