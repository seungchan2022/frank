## 목적

즐겨찾기(스크랩) 기사 탭과 오답 노트 탭에 피드와 동일한 태그 칩 필터 UI를 추가한다.
콘텐츠가 누적될수록 특정 태그로 빠르게 탐색할 수 있어야 한다.

---

## 인터뷰 확정 결정

| 항목 | 결정 | 근거 |
|---|---|---|
| FavoritesFeature 태그 의존성 | **B안**: TagPort 주입 | FeedFeature 패턴과 일치, FavoriteItem.tagId 기반 tags 필터링 |
| WrongAnswers 태그맵 | **A안**: FavoritesView 레벨에서 feature.items 계산 | Feature 간 결합 없음, WrongAnswer.articleUrl ↔ FavoriteItem.url 조인으로 tagId 매핑 |
| WrongAnswer 모델 변경 | **없음** | 모델 변경 불필요, 뷰 레벨 조인으로 충분 |
| 서버 변경 | **없음** | tagId는 이미 FavoriteItem 반환값에 포함 |

---

## 입력 / 출력

### 입력

- `FavoriteItem.tagId: UUID?` (기존 서버 응답)
- `WrongAnswer.articleUrl: String` (기존 서버 응답)
- `TagPort.fetchAllTags() → [Tag]` (기존 포트 재사용)
- `TagPort.fetchMyTagIds() → [UUID]` (기존 포트 재사용)

### 출력

- FavoritesView 기사 탭: 가로 스크롤 태그 칩 바 + 선택 시 클라이언트 필터링
- FavoritesView 오답 탭: 가로 스크롤 태그 칩 바 + 선택 시 클라이언트 필터링
- 필터 해제 시 전체 목록 복원

---

**테스트**: `FavoritesFeatureTests.swift`에 태그 필터 케이스 추가
- `selectTag(nil)` → `filteredItems` 전체 반환
- `selectTag(tagId)` → 해당 tagId 아이템만 반환
- `selectTag(nil)` 재선택 → 전체 복원

### ST-3: WrongAnswersView에 태그 칩 필터 상태 추가

**파일**: `ios/Frank/Frank/Sources/Features/Favorites/FavoritesView.swift`

**변경 내용**:
- `FavoritesView`에 `@State private var wrongAnswersSelectedTagId: UUID?` 추가
- `wrongAnswersTags: [Tag]` computed 프로퍼티 추가
  - `feature.items`에서 `tagId`가 있는 아이템 → `feature.tags`에서 해당 Tag 조회 → 중복 제거
  - (WrongAnswer.articleUrl ↔ FavoriteItem.url 조인 → tagId 추출)
- `filteredWrongAnswers: [WrongAnswer]` computed 프로퍼티 추가
  - `wrongAnswersSelectedTagId == nil` → 전체
  - `wrongAnswersSelectedTagId != nil`:
    1. `feature.items`에서 해당 tagId 아이템의 url Set 추출
    2. `wrongAnswersFeature.items.filter { urlSet.contains($0.articleUrl) }`

> **아키텍처 경계 주의 (Codex 리뷰)**: `wrongAnswersTags` / `filteredWrongAnswers`는 표현 로직(뷰 레벨)에 해당하므로 `FavoritesView` 내 private computed 프로퍼티로 허용.
> 단, 이 로직이 복잡해지면 별도 테스트 가능한 순수 함수(`static func`)로 분리하는 것을 고려.

**테스트 (Specify 단계)**:
- `FavoritesViewTests.swift` — 뷰 레벨 computed 필터 로직에 대한 단위 검증
  - `wrongAnswersTags`: FavoriteItems에 tagId가 있을 때 해당 Tag만 반환
  - `filteredWrongAnswers(nil)`: 전체 WrongAnswer 반환
  - `filteredWrongAnswers(tagId)`: url Set 교집합 기반 필터 결과 확인
- `WrongAnswersFeatureTests.swift` — 기존 테스트 전체 그린 유지
