# MVP11 M3 메인태스크: 웹 즐겨찾기·오답 태그 필터

## 분석 요약

### 페이지 구조 확인
- **오답 노트 별도 라우트 없음**: `/routes/favorites/+page.svelte` 단일 파일 안에 세그먼트 탭(`activeTab: 'articles' | 'wrong-answers'`)으로 구현되어 있음. 기획 리스크(오답 페이지 미구현)는 해소됨 — 즐겨찾기·오답 탭 모두 이 파일 하나에서 처리
- **태그 칩 컴포넌트 없음**: `feed/+page.svelte`에 태그 필터 UI가 인라인으로 구현되어 있음. 재사용 가능한 `TagChips.svelte` 컴포넌트 없음 → 추출 필요

### 태그 데이터 구조
- `Favorite.tagId: string | null` — 기사 탭 필터링에 사용 가능
- `WrongAnswer` 타입에는 `tagId` 없음 (`articleUrl`, `articleTitle`만 존재)
  - 오답 탭 태그 필터: `wrongAnswers`에서 `articleUrl`을 키로 `favorites` 목록과 대조해 tagId 매핑 필요 (favoritesStore는 이미 로드된 상태로 활용)
- `feedStore.tags`에 전체 태그 목록이 있으나 즐겨찾기 페이지는 `feedStore` 미사용
  - 태그 목록 조회 전략: `apiClient.fetchTags()` 직접 호출 또는 `feedStore` import 두 가지 옵션 → `feedStore` import가 가장 단순 (이미 auth/$effect에서 로드 보장)

### 현재 feedStore 태그 필터 패턴 (재사용 대상)
```svelte
<!-- feed/+page.svelte 인라인 패턴 -->
```

- 즐겨찾기 페이지에서는 feedStore 의존 없이 로컬 `$state`로 동일 패턴 구현

## 구현 순서

1. **ST-1: TagChips 컴포넌트 추출** — `feed/+page.svelte`의 인라인 태그 필터 UI를 `TagChips.svelte`로 분리. Props: `tags: Tag[]`, `activeTagId: string | null`, `onSelect: (id: string | null) => void`

2. **ST-2: 즐겨찾기 기사 탭 태그 필터 추가** — `favorites/+page.svelte`에 로컬 상태 추가:
   ```svelte
   let selectedTagId = $state<string | null>(null);
   let availableTags = $derived([...new Map(
     favoritesStore.favorites
       .filter(f => f.tagId)
       .map(f => [f.tagId, { id: f.tagId!, name: tagName(f.tagId!) }])
   ).values()]);
   let filteredFavorites = $derived(
     selectedTagId
       ? favoritesStore.favorites.filter(f => f.tagId === selectedTagId)
       : favoritesStore.favorites
   );
   ```
   - 태그 이름 조회: `feedStore` import 후 `feedStore.tags`에서 이름 매핑 (feedStore는 feed 페이지 방문 후 이미 로드됨). 또는 `apiClient.fetchTags()` 직접 호출 (feedStore 미의존)
   - 탭 전환 시 `selectedTagId = null` 초기화 (상태 전이 완결성 규칙)

3. **ST-3: 오답 탭 태그 필터 추가** — `WrongAnswer`에 tagId 없으므로 `articleUrl → tagId` 매핑:
   ```svelte
   // favoritesStore.favorites 로드 보장 후 매핑
   let urlToTagId = $derived(
     new Map(favoritesStore.favorites.map(f => [f.url, f.tagId]))
   );
   let filteredWrongAnswers = $derived(
     selectedWrongTagId
       ? wrongAnswers.filter(wa => urlToTagId.get(wa.articleUrl) === selectedWrongTagId)
       : wrongAnswers
   );
   ```
   - 즐겨찾기에 없는 오답(tagId 매핑 불가)은 "전체" 탭에는 표시, 특정 태그 필터 시 제외
   - 오답 탭 전용 `selectedWrongTagId` 상태 분리 (기사 탭과 독립)

4. **ST-4: TagChips 컴포넌트 단위 테스트** — vitest + @testing-library/svelte로 렌더링 및 이벤트 검증

5. **ST-5: 즐겨찾기 필터 로직 단위 테스트** — 순수 함수 추출 후 필터링 로직 검증 (기존 favorites-quiz-button.test.ts 패턴 동일)

6. **ST-6: 전체 검증** — `npm run validate` 통과 + Chrome DevTools/Playwright 스크린샷

## 주의사항

- `feed/+page.svelte`의 태그 필터 코드도 ST-1 작업 시 TagChips 컴포넌트로 교체하여 일관성 유지 (선택사항 — 범위 확장 여부 구현 시 판단)
- 탭 전환(`activeTab` 변경) 시 각 탭의 `selectedTagId`를 초기화해야 이전 탭 필터 상태가 오염되지 않음 (Svelte 5 상태 전이 완결성 규칙)
- `WrongAnswer.articleUrl`과 `Favorite.url` 매핑은 사용자가 즐겨찾기를 해제한 경우 tagId가 null이 될 수 있음 — "전체" 처리로 안전하게 처리
